# v0.2 Phase B — Push Notifications

## Context

Phase A landed the PWA installable shell. Phase B is "the point of the
project": when an agent needs attention (permission prompt, task done,
error, crash), the user's phone buzzes with a push notification. Tapping
the notification opens the PWA at the relevant agent.

Almost all infrastructure already exists:
- DB tables: `alerts`, `push_subscriptions` (migration 001)
- Query functions: `insertAlert`, `listRecentAlerts`, `acknowledgeAlert`,
  `upsertPushSub`, `listPushSubsForUser`, `deletePushSubByEndpoint`
- WS protocol: `SC_Alert` message type defined in `protocol.ts:146-154`
- Config: `vapidPublicKey`, `vapidPrivateKey`, `vapidSubject` loaded in
  `config.ts:52-54,83-85`
- Dependency: `web-push@^3.6.7` + `@types/web-push` in package.json
- `.env.example` documents VAPID key generation (lines 29-34)
- Service worker: Phase B extension points at `src/service-worker.ts:112-113`
- Event pipeline: `AgentRuntime.onChunk()` already calls `insertEvent()`
  and emits `event`/`state` over WebSocket

What's missing is the glue: alert creation, push fan-out, subscription
endpoints, client-side registration, service worker push/click handlers,
and a notification preferences UI.

## Implementation Steps

### 1. Push notification service — `src/lib/server/push/PushService.ts` (new)

Singleton that wraps `web-push`. Initialised in `bootstrap.ts` alongside
the supervisor.

```typescript
import webpush from 'web-push';
import { getConfig } from '../config.js';
import { listPushSubsForUser, deletePushSubByEndpoint } from '../db/queries.js';

class PushService {
  private ready = false;

  init(): void {
    const cfg = getConfig();
    if (!cfg.vapidPublicKey || !cfg.vapidPrivateKey) {
      console.warn('[push] VAPID keys not set — push disabled');
      return;
    }
    webpush.setVapidDetails(cfg.vapidSubject, cfg.vapidPublicKey, cfg.vapidPrivateKey);
    this.ready = true;
  }

  async notifyUser(userId: string, payload: PushPayload): Promise<void> {
    if (!this.ready) return;
    const subs = listPushSubsForUser(userId);
    const json = JSON.stringify(payload);
    await Promise.allSettled(
      subs.map(async (sub) => {
        try {
          await webpush.sendNotification(
            { endpoint: sub.endpoint, keys: { p256dh: sub.p256dh, auth: sub.auth } },
            json,
            { TTL: 60 * 5 }
          );
        } catch (err: any) {
          // 404/410 = subscription expired → clean up
          if (err.statusCode === 404 || err.statusCode === 410) {
            deletePushSubByEndpoint(sub.endpoint);
          }
        }
      })
    );
  }
}

interface PushPayload {
  title: string;
  body: string;
  data: {
    agentId: string;
    alertId: string;
    // deep-link the PWA opens on tap
    url: string;
  };
}
```

Export a singleton getter `getPushService()` like the existing
`getSupervisor()` pattern in `bootstrap.ts`.

### 2. Alert pipeline — hook into `AgentRuntime.onChunk()`

**File: `src/lib/server/agents/AgentRuntime.ts`** — after `insertEvent()`
and the state-change block (~line 193), add alert creation + push:

```typescript
import { insertAlert, listRecentAlerts } from '../db/queries.js';
import { getPushService } from '../bootstrap.js';
import { ulid } from 'ulid';

// Inside the event loop, after status updates:
if (shouldAlert(ev)) {
  // Dedup: skip if an unacknowledged alert for same (agent, patternId)
  // exists within the last 30 seconds
  const recent = listRecentAlerts(this.agent.id, Math.floor(ev.at / 1000) - 30);
  const isDupe = recent.some(
    (a) => JSON.parse(a.payload_json).patternId === ev.patternId && !a.acknowledged_at
  );
  if (!isDupe) {
    const alertId = ulid();
    const reason = alertReason(ev);
    insertAlert({
      id: alertId,
      user_id: this.agent.user_id,
      agent_id: this.agent.id,
      severity: ev.severity ?? 'info',
      reason,
      payload_json: JSON.stringify({ patternId: ev.patternId, choices: ev.choices, detail: ev.detail }),
      ts: Math.floor(ev.at / 1000)
    });

    // Fan out to all WS subscribers
    this.emit('alert', { id: alertId, severity: ev.severity ?? 'info', reason, ev });

    // Push notification to phone
    getPushService().notifyUser(this.agent.user_id, {
      title: `${this.agent.cli_kind}: ${reason}`,
      body: alertBody(ev, this.agent),
      data: { agentId: this.agent.id, alertId, url: `/repos/${this.agent.repo_id}?agent=${this.agent.id}` }
    });
  }
}
```

**`shouldAlert()` rules:**
- `prompt_detected` → always (this is the primary use case)
- `task_done` → always
- `error` → always
- `exited` → alert via supervisor exit handler, not here
- `ready`, `working` → never

**`alertReason()` helper:**
- `prompt_detected` → `"Permission needed"` (or include tool name from detail)
- `task_done` → `"Task complete"`
- `error` → `"Error: <patternId>"` (e.g., "Error: rate_limit")

### 6. Client-side push registration

**File: `src/lib/client/push.ts`** (new)

Utility module called from the layout on mount:

```typescript
export async function registerPush(vapidPublicKey: string): Promise<void> {
  if (!('serviceWorker' in navigator) || !('PushManager' in window)) return;
  if (Notification.permission === 'denied') return;

  const permission = await Notification.requestPermission();
  if (permission !== 'granted') return;

  const reg = await navigator.serviceWorker.ready;
  let sub = await reg.pushManager.getSubscription();
  if (!sub) {
    sub = await reg.pushManager.subscribe({
      userVisibleOnly: true,
      applicationServerKey: urlBase64ToUint8Array(vapidPublicKey)
    });
  }
  const keys = sub.toJSON().keys!;
  await fetch('/api/push/subscribe', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      endpoint: sub.endpoint,
      keys: { p256dh: keys.p256dh, auth: keys.auth }
    })
  });
}
```

### 8. Service worker: push + notificationclick handlers

**File: `src/service-worker.ts`** — replace the Phase B comments:

```typescript
self.addEventListener('push', (event) => {
  if (!event.data) return;
  const payload = event.data.json() as {
    title: string;
    body: string;
    data: { agentId: string; alertId: string; url: string };
  };
  event.waitUntil(
    self.registration.showNotification(payload.title, {
      body: payload.body,
      icon: '/icons/icon-192.png',
      badge: '/icons/icon-192.png',
      tag: `maw-${payload.data.agentId}`,
      renotify: true,
      data: payload.data
    })
  );
});

self.addEventListener('notificationclick', (event) => {
  event.notification.close();
  const url = event.notification.data?.url ?? '/';
  event.waitUntil(
    self.clients.matchAll({ type: 'window', includeUncontrolled: true }).then((clients) => {
      // Focus existing MAW tab if open
      for (const client of clients) {
        if (new URL(client.url).origin === self.location.origin) {
          client.focus();
          client.navigate(url);
          return;
        }
      }
      // Otherwise open a new window
      return self.clients.openWindow(url);
    })
  );
});
```

## Dependencies on other plans

- **v0.2-configurable-spawn-args.md** — if the user has
  `--dangerously-skip-permissions` enabled (the default for them),
  `prompt_detected` events won't fire and those push notifications
  won't trigger. This is expected and correct — the setting controls
  whether the CLI even asks. Push notifications for `task_done`,
  `error`, and `exited` still work regardless.
