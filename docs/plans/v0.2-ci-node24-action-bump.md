# Fix CI: bump actions to Node 24 runtime (vitest workflow deprecation)

## Context

The `vitest` job in `.github/workflows/unit.yml` surfaces this banner on every run:

> Node.js 20 actions are deprecated. The following actions are running on Node.js 20 and may not work as expected: `actions/checkout@v4`, `pnpm/action-setup@v4`. Actions will be forced to run with Node.js 24 by default starting June 2nd, 2026.

The user reads this as a unit-test failure, but the unit suite itself is green:

```
Test Files  49 passed (49)
Tests       767 passed (767)
Duration    2.31s
```

(Verified locally on the worktree with the project's own `pnpm test --project=server --project=client`.)

The project **already runs on Node.js 24** — `actions/setup-node@v6` with `node-version: 24` (unit.yml:20-23, e2e.yml:25-28). The deprecation warning is about the **action wrappers' own bundled JavaScript runtime**, not the project's Node version. `actions/checkout@v4` and `pnpm/action-setup@v4` both still ship `using: node20` in their `action.yml` (verified via the GitHub API).

The fix is to bump those two pinned majors to a release that ships `using: node24`:

| Action | Current | Latest stable | First Node-24 release |
|---|---|---|---|
| `actions/checkout` | v4 | v6.0.2 (2026-01-09) | v5.0.0 (2025-08-11) |
| `pnpm/action-setup` | v4 | v6.0.5 (2026-05-02) | v5.0.0 (2026-03-17) |
| `actions/setup-node` | v6 ✓ | v6.4.0 | already current |
| `actions/upload-artifact` | v7 ✓ | v7.x | already current |

Breaking-change check (release notes):
- **`actions/checkout` v4 → v6**: no changes to default-usage surface. v5 was the Node 24 bump, v6 added "persist creds to a separate file" (internal). Our call site is parameterless — safe.
- **`pnpm/action-setup` v4 → v6**: v5 was the Node 24 bump, v6 added pnpm 11 support. We pin `version: 10` explicitly, so the default doesn't drift — safe.

## Plan

Bump two action references in two workflow files. Identical edits in both.

### Files to modify

1. **[.github/workflows/unit.yml](.github/workflows/unit.yml)** — lines 14 and 16
2. **[.github/workflows/e2e.yml](.github/workflows/e2e.yml)** — lines 14 and 21

### Exact edits

In both files, replace:

```yaml
      - uses: actions/checkout@v4
```

with:

```yaml
      - uses: actions/checkout@v6
```

And replace:

```yaml
      - uses: pnpm/action-setup@v4
        with:
          version: 10
```

with:

```yaml
      - uses: pnpm/action-setup@v6
        with:
          version: 10
```

(Keep the explicit `version: 10` pin — v6 added pnpm 11 support but doesn't force it; we want stability.)

No source code, no `package.json`, no `vitest.config.ts` changes.

3. **Rollback** — if either action surfaces a regression, the diff is two-line per file: revert via `git revert` or pin to v5 instead (also Node 24 capable, smaller jump).

## Why not the env-var workaround

The warning suggests `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true` as an opt-in. We're rejecting it because:
- It's a transitional flag — GitHub will flip the default on 2026-06-02 anyway, then remove Node 20 from runners on 2026-09-16.
- Bumping the pin is the durable fix: it survives the transition, removes the warning *now*, and matches the rest of our actions (setup-node@v6, upload-artifact@v7) which are already on current majors.

## Out of scope

- Migrating to pnpm 11 (we'd want to do that as its own change with a lockfile bump and CI run).
- Touching `actions/setup-node` or `actions/upload-artifact` — both already on current majors.
- Any vitest config or test changes — the suite is healthy.
