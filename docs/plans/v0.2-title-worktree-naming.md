# Mandatory agent titles, title-based worktree naming, drop projects from spawn dialog

## Context

Today the spawn dialog has three friction points:

1. **Task title is optional.** Stored on `tasks` (`tasks.title`), joined into agent rows via `agents.current_task_id`. Agents without a task title render as `projectName/<nothing>` and worktree directories are opaque ULIDs — hard to tell apart on the filesystem.
2. **Worktree dir = ULID.** `WorktreeManager.create()` at [src/lib/server/git/WorktreeManager.ts:61](src/lib/server/git/WorktreeManager.ts:61) does `join(worktreeRoot, opts.agentId)`, so paths like `~/.local/share/maw/worktrees/01KP8BCFWM…/` carry no semantic meaning.
3. **Projects layer is in the way.** The only project UI left in the spawn flow is the nested "Project selector + New project" block inside the inline-repo-creator in [SpawnAgentForm.svelte](src/lib/client/components/SpawnAgentForm.svelte) (~lines 362–410). The user no longer wants to model projects — repos should be the top-level entity.

Goal: require a title at spawn time, use a slug of the title as both the worktree directory name and the uniqueness key, and drop the project picker/creator from the spawn dialog (making `repos.project_id` nullable so new inline repos can be created without one).

## Changes

### 1. DB migration — `migrations/004_repo_project_optional.sql` (new)

SQLite doesn't support `ALTER COLUMN`, so rebuild `repos`:

- Create `repos_new` with `project_id TEXT REFERENCES projects(id) ON DELETE SET NULL` (nullable) and a new `default_branch TEXT NOT NULL DEFAULT 'main'` column.
- `INSERT INTO repos_new SELECT r.id, r.user_id, r.project_id, r.path, r.origin_url, COALESCE(p.default_branch, 'main'), r.created_at, r.updated_at FROM repos r LEFT JOIN projects p ON p.id = r.project_id;`
- Drop old `repos`, rename, recreate `idx_repos_project`.

### 2. Slug utility — `src/lib/server/util/slug.ts` (new)

```ts
export function slugifyTitle(title: string): string {
  const s = title.toLowerCase().normalize('NFKD')
    .replace(/[\u0300-\u036f]/g, '')
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-+|-+$/g, '')
    .slice(0, 60);
  return s;
}
```

Empty result → caller treats as validation error ("title must contain at least one letter or digit").

### 3. Spawn server action — [src/routes/agents/new/+page.server.ts](src/routes/agents/new/+page.server.ts)

- Require `task_title` (trimmed, non-empty). Return `fail(400, ...)` if missing.
- `const slug = slugifyTitle(task_title);` reject if empty.
- Duplicate check: query `worktrees` for any row (active, orphaned, or removed) whose `path` equals `join(cfg.worktreeRoot, slug)`. Add `findWorktreeByPath(path)` to [queries.ts](src/lib/server/db/queries.ts). Also safety-net check `fs.existsSync(targetPath)`. If either hits → `fail(409, { error: 'title already used' })`.
- Keep `agentId = ulid()` and `branch = maw/<agentId>` (ULID guarantees git-branch uniqueness even if titles are later edited or reused post-deletion).
- Pass a new `dirName: slug` field into `WorktreeManager.create()`.
- Task row is still created from title+body; title is now guaranteed present.

### 4. `WorktreeManager.create()` — [src/lib/server/git/WorktreeManager.ts](src/lib/server/git/WorktreeManager.ts:55)

Signature change:
```ts
create(opts: { repoPath: string; agentId: string; branch: string; startPoint: string; dirName?: string }): Promise<string>
```
Line 61 becomes `const wtPath = join(this.worktreeRoot, opts.dirName ?? opts.agentId);`. If `wtPath` already exists on disk, throw — the spawn server already checked, this is defense in depth.

### 5. Spawn form — [src/lib/client/components/SpawnAgentForm.svelte](src/lib/client/components/SpawnAgentForm.svelte)

- Mark `task_title` input `required`; add live helper text showing the computed slug preview so users see the directory name.
- Delete the nested "Project selector + New project" block (lines ~362–410, plus the `createProject`, `showNewProject`, `newProjectName`, `newProjectBranch`, `projectOptions` state and the `projects` prop).
- Inline "new repo" POST to `/api/repos` drops `project_id`; optionally sends `default_branch` (defaults to `'main'` server-side).
- Repo dropdown label falls back to `r.path` when `projectName` is empty.

### 6. `POST /api/repos` — [src/routes/api/repos/+server.ts](src/routes/api/repos/+server.ts)

- `project_id` becomes optional. If absent: skip the `getProject` lookup, use `default_branch` from request body (default `'main'`), insert repo with `project_id = null`.
- `insertRepo` in [queries.ts](src/lib/server/db/queries.ts) accepts nullable `project_id` and a `default_branch` field.

### 7. Null-project rendering fallbacks

- `listReposWithProjectForUser` / `AgentCardRow` in [queries.ts](src/lib/server/db/queries.ts) already `LEFT JOIN` projects — confirm they tolerate null `p.name` and `p.default_branch`.
- [AgentCard.svelte:191](src/lib/client/components/AgentCard.svelte:191): when `project_name` is null, show just `task_title` (no leading `/`).
- Sidebar tree ([RepoTreeSidebar.svelte](src/lib/client/components/RepoTreeSidebar.svelte) and its layout loader): render repos without a project under the same **Repositories** root; no new UI needed since the tree is already flat at repo level.
- `/agents/new` loader: drop the `projects` prop.

### 8. Out of scope (no change)

- Existing ULID-named worktree directories keep working — `wt.path` is stored per worktree, so JSONL history reconstruction ([ClaudeJsonlHistory.ts:33-38](src/lib/server/agents/history/ClaudeJsonlHistory.ts:33)) and archive pages resolve them unchanged.
- Branch naming stays `maw/<agentId>` — renaming branches would break the supervisor reattach path.
- Project CRUD pages/routes are not touched; projects aren't removed, just demoted out of the spawn flow.
