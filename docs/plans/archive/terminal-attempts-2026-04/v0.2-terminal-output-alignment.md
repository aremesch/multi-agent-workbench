# v0.2 — Terminal Output Alignment Fixes

## Context

Two output/cursor misalignment bugs degrade the terminal viewer:

**Bug #1 — Bash agent, extra line feeds after the prompt.**
After spawning a `shell.jsonc` agent (`scrollbackMode: 'history'`), the xterm cursor
lands two rows below the bash prompt instead of at the end of `bash-5.2$ `. Keystrokes
visually appear on the wrong row.

Root cause, verified by tracing the snapshot path:
1. `tmux capture-pane -S -500 -p -e` returns one line per pane grid row — for a fresh
   bash pane that's the prompt on row 0 plus many blank rows below (`src/lib/server/tmux/TmuxSession.ts:146`).
2. `collapseRepeatingTailBlocks` (`src/lib/server/ws/hub.ts:421`) greedily halves
   repeated tail blocks but only for `k ≥ 2`, so it can never fully drain a run of
   trailing empties — after convergence there are always 2–3 blank lines left.
3. The snapshot is CRLF-normalized and written into a freshly-`reset()`
   xterm (`src/lib/client/components/AgentTerminalPanel.svelte:80`). Xterm's cursor
   ends wherever the last character lands, i.e. several rows below the prompt.
4. Neither side carries the actual tmux cursor position, so there's nothing to
   correct the landing row.

**Bug #2 — Claude Code, TUI drifts between open and reopen.**
On first open the TUI is anchored at the bottom of the viewport (correct). After
closing and reopening the modal, content renders at the top with the cursor dangling
below empty rows — new output from Claude arrives visually detached from the
previous turn.

Root cause:
1. For `scrollbackMode: 'visible'` the snapshot is `capture-pane -S 0` (current
   screen) without dedup. Whatever rectangle tmux has in the grid gets shipped —
   including blank trailing rows when Claude's TUI is top-aligned after a repaint
   (`src/lib/server/ws/hub.ts:194`).
2. Same cursor-position gap as Bug #1: xterm's cursor lands at the bottom of the
   blank-padded frame, not at Claude's real cursor position inside the TUI.
3. The preceding `history_snapshot` (Claude JSONL, written before the live frame in
   `AgentTerminalPanel.svelte:81-84`) consumes rows of xterm buffer, so the blank
   rows pad the frame even further away from the real cursor.
4. Claude's subsequent differential output (CUP-based) lands where Claude thinks its
   cursor is, while any plain `\r\n` output appends at the detached xterm cursor —
   hence the "new text appears detached" behaviour.

Goal: make the reconnect snapshot land the xterm cursor where tmux's cursor actually
is, regardless of adapter. Lock the behaviour down with Playwright regressions.

## Approach

### 1. Trim trailing blanks *before* dedup, then always
**File:** `src/lib/server/ws/hub.ts` (`sendReconnectSnapshot`, around lines 225–239).

- Before invoking `collapseRepeatingTailBlocks`, rstrip the raw capture of trailing
  lines whose visible content (ANSI-stripped via the already-installed `strip-ansi`
  package) is empty. This makes the dedup operate only on real content and removes
  the "2 empty lines always survive" artefact that causes Bug #1.
- Apply the same rstrip in `'visible'` mode (no dedup there, but trailing blanks
  still push the cursor down in Bug #2).

### 2. Append a cursor-position CSI to the snapshot
**File:** `src/lib/server/tmux/TmuxSession.ts`
- Add `Tmux.cursorPosition(session): Promise<{ x: number; y: number } | null>`
  wrapping `tmux display-message -p -t <session> '#{cursor_x},#{cursor_y}'`. Returns
  `null` if the session is gone or the response fails to parse. Coordinates are
  0-indexed (tmux default, verified: `0,0` for fresh pane, `10,0` after bash prompt).

**File:** `src/lib/server/ws/hub.ts` (`sendReconnectSnapshot`)
- After the CRLF-normalized snapshot body, query `Tmux.cursorPosition(runtime.tmuxSession)`
  and append `\x1b[<y+1>;<x+1>H` (CSI CUP, 1-indexed for the wire). Apply to both
  `'visible'` and `'history'` modes.
- Clamp coordinates to the viewer's `cols`/`rows` if provided, so a cursor that tmux
  places outside the current viewport (e.g. post-shrink) doesn't land off-screen.

### 3. A/B: also force a Claude-side redraw on reopen?
The user asked me to try both approaches and empirically pick the best. For
`scrollbackMode: 'visible'` specifically, compare:

- **Variant A — cursor-sync only** (steps 1+2). Minimal, zero extra redraws.
- **Variant B — cursor-sync + forced redraw.** On subscribe, after the snapshot is
  sent, call `Tmux.resizeWindow(session, cols ± 1, rows)` followed by `(cols, rows)`
  so the CLI sees a SIGWINCH even when dims are unchanged. This bypasses the
  no-op guard at `AgentRuntime.resize` (`src/lib/server/agents/AgentRuntime.ts:149`).

Decision procedure: run the Playwright `terminal-claude-reopen` spec against both
variants, pick whichever produces stable cursor alignment without visible flicker.
Gate the forced redraw behind a per-adapter flag (`forceRedrawOnReconnect?: boolean`
on the adapter descriptor) so we only enable it where it actually helps.

If Variant A passes the spec cleanly, drop Variant B before committing. Otherwise
keep both variants but default the flag to `true` only on `claude-code.jsonc`.

### 4. No change to `history_snapshot` semantics
The Claude JSONL prepend path (`AgentTerminalPanel.svelte:66-86`) keeps working: the
cursor-position CSI at the end of the live frame wins over whatever position the
history body left the cursor in, so prepending history no longer matters for
alignment.

### 5. Playwright E2E regression suite
No Playwright setup exists today — add it from scratch alongside the fix. Per the
user's scope decision, the suite is narrow: one spec per bug, no reconnect/resize
coverage this round.

- **Dep:** add `@playwright/test` as a devDependency (`package.json`). Add `test:e2e`
  script: `playwright test`.
- **Config:** new `playwright.config.ts` — single browser (chromium), `baseURL`
  from `process.env.MAW_E2E_URL ?? 'http://emaw:3000'`, reuses auth via
  `storageState` produced by a setup project.
- **Helpers under `tests/e2e/helpers/`:**
  - `auth.ts` — logs in by POSTing the `?/login` form action with credentials from
    `MAW_E2E_USERNAME` / `MAW_E2E_PASSWORD` env vars, captures the `maw_session`
    cookie, persists it as storage state for reuse.
  - `terminal.ts` — reads the xterm `Terminal` instance attached to the active
    panel via `page.evaluate`; returns `{ cursorX, cursorY, bufferLines }` derived
    from `term.buffer.active.cursorX/cursorY` and `term.buffer.active.getLine(i).translateToString(true)`.
    Also exposes `waitForLine(page, re)` for text matching.
  - `agent.ts` — spawn an agent through the existing form-action pipeline (project,
    repo, role, spawn), return its `id`. Also `openAgentModal(page, id)` and
    `closeAgentModal(page)`.
- **Specs under `tests/e2e/`:**
  - `terminal-bash-cursor.spec.ts` — spawn a `shell` agent, open the modal, wait
    for the prompt text, assert `cursorY === <row of prompt>` and the cursor sits
    immediately after the `$ ` glyph. Regression for Bug #1.
  - `terminal-claude-reopen.spec.ts` — spawn a `claude-code` agent (skip the whole
    test via `test.skip` if `claude` is not on PATH server-side; detect with a
    `/health`-like probe or a harmless API call). Send a short prompt, wait for the
    first reply. Close the modal, reopen. Assert the xterm cursor row is within the
    last rendered TUI footer row ± 1, not dangling below blank rows. Regression for
    Bug #2.
- Expose a test-only window handle for the shared `MawWsClient` so any future test
  can drive reconnects (`src/lib/client/ws.ts`): if `import.meta.env.DEV`, set
  `(window as any).__maw_ws_for_test = singleton` in `getMawWsClient()`. Production
  bundles skip the assignment.
