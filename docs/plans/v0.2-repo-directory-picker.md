# Plan: Directory picker for inline repo creation

## Context

Today, when a user creates a repo inline from the spawn-agent dialog
(`SpawnAgentForm.svelte`), they must type the absolute server-side
path to the repo directory into a plain `<input>`. This is awkward on
desktop and unusable on the mobile PWA (the phone has no idea what
paths exist on the MAW server). Mistyped paths are the most common
spawn-flow error.

Add a **Browse** button next to the path input that opens a modal
directory picker — breadcrumb trail + single-column list of
subdirectories, with a "Select this directory" confirm button. The
picker is sandboxed to the server user's `$HOME` so the new list API
doesn't expose `/etc`, `/var`, etc. The existing `POST /api/repos`
path validation (absolute, exists, isDirectory) is untouched; the
picker simply fills the input with an absolute path under `$HOME`.

Scope (from clarifying questions):

- **Browse scope:** Sandboxed to `$HOME`. Rejects symlink escapes via
  `realpath`.
- **Entry point:** Only the inline creator in
  `SpawnAgentForm.svelte`. The legacy standalone page
  (`/projects/[id]/repos/new`) and `RepoEditDialog` are **not**
  changed.
- **UI:** Modal, breadcrumb + single-column list. Mobile-first.
  Hidden dirs (`.foo`) hidden by default with a toggle. Directories
  that are themselves git repos get a small "git" marker.

## API contract

`GET /api/fs/list?path=<abs>&hidden=<0|1>`

- Auth: 401 if no `locals.user`.
- `path` omitted ⇒ defaults to sandbox root (`$HOME`).
- Server steps:
  1. `resolve(path)` (absolute-ify).
  2. `realpathSync(resolved)` to collapse symlinks; catch ENOENT → 404.
  3. Require `real.startsWith(rootReal + sep) || real === rootReal`;
     otherwise 403 `picker.error.outsideRoot`.
  4. `statSync` must be a directory; else 400.
  5. `readdirSync(real, { withFileTypes: true })`:
     - Skip non-directories.
     - Skip dotdirs unless `hidden=1`.
     - For each, compute `isGitRepo = existsSync(join(real, name, '.git'))`.
  6. Return
     `{ path: real, parent: real === rootReal ? null : dirname(real), entries: [{name, isGitRepo}] }`,
     sorted by `name` (locale-aware, case-insensitive).

No stat on entries beyond the single `.git` existence check —
directory listings on a large home can otherwise be slow.

## Security notes

- `realpath` after resolve is what closes the symlink-escape hole; a
  raw string prefix check isn't enough.
- The sandbox root itself is also `realpathSync`'d once at startup
  (cached) so a symlinked `$HOME` doesn't produce false negatives.
- No path input from the client is ever passed to a shell command —
  everything stays in `node:fs` / `node:path`.
- GET is fine without CSRF; it's the existing pattern for read-only
  routes (see `hooks.server.ts:21`).

## UX details

- Modal title: `picker.title` ("Select directory").
- Header row: breadcrumb (clickable segments) with
  `/home/<user>` rendered as `~`. Truncates from the left on narrow
  viewports (keep the tail visible — that's where the user is).
- Body: one `<button>` per child dir, full-width, `py-3` tap target
  for thumbs. Right side shows the `picker.gitRepo` chip when
  applicable.
- Footer:
  - Toggle: "Show hidden" (persisted to `localStorage`
    `maw.picker.showHidden`).
  - Primary: "Select this directory" → calls `onSelect(currentPath)`
    and closes. Disabled at the sandbox root only if that would be a
    nonsensical pick (leave enabled — some users' `$HOME` itself
    might be a git repo).
- Empty dir: shows `picker.empty` centered.
- Errors: inline red banner at the top of the body, preserves the
  last successful listing so the user can retry or navigate up.
- Autofocus lands on the first directory button so keyboard users
  can arrow through.

  (`RepoEditDialog.svelte`, `SpawnAgentForm.svelte`).

No change to:

- `POST /api/repos` validation (still checks absolute + exists +
  isDirectory — picker just produces inputs that pass).
- `WorktreeManager.initEmpty` / `ensureDefaultBranch` self-healing
  (`src/lib/server/git/WorktreeManager.ts:146`).
- DB schema or `listReposWithProjectForUser`.

## Verification

1. `pnpm check` clean.
2. `pnpm test` — new unit tests for `browse.ts` pass (sandbox clamp,
   symlink escape rejection, hidden filter, git-repo marker,
   root-parent null).
3. `pnpm test:e2e` — existing suite still green; no new e2e spec
   required (covered by unit tests + manual PWA check below).
4. Manual, on the dev server:
   - Dashboard → spawn FAB → **New Repo** inline → **Browse** opens
     the modal at `~`.
   - Navigate into a subdirectory, breadcrumb updates, Up climbs
     back.
   - Try to fetch `/api/fs/list?path=/etc` directly → 403.
   - `ln -s /etc ~/escape && curl … ?path=~/escape` → 403 after
     `realpath`.
   - Pick a directory that is an empty dir → spawn agent → worktree
     `git init` self-heal fires as expected.
   - Pick a directory that already has a git repo with legacy
     `master` → confirm `ensureDefaultBranch` renames to `main`.
5. Mobile/PWA check: install the PWA on a phone, exercise the
   picker with a thumb — breadcrumb truncation, tap targets, hidden
   toggle all feel right.

## Out of scope (explicit)

- Not touching the legacy `projects/[id]/repos/new` page.
- No file picker (directories only — MAW attaches repos by
  directory, never a file).
- No multi-root config or `MAW_REPO_ROOTS` env. If later desired,
  flip `getFsBrowseRoot()` to return a list and have the picker show
  the roots as the top level.
- No creation of directories from within the picker (user can
  always type a path manually if they need to create one).
