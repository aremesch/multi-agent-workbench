# Playwright hydration smoke tests

## Context

A CSP regression (commit 273f95b) silently broke client-side hydration on
every page: `script-src 'self'` without a matching hash blocked SvelteKit's
inline bootstrap script. SSR output still rendered, but no `onclick`
handler ever attached — the user menu and the + FAB appeared to do nothing.
`svelte-check` and `vitest` were blind to it because the failure is a
browser policy violation, not a type or unit error. The bug sat on
`development` for three days before a user noticed.

The lesson: nothing in the current test pipeline exercises the real server,
the real CSP, and a real browser. A tiny browser-level smoke suite closes
that gap and catches the whole class of "SSR renders, hydration dies"
regressions — CSP, COOP/COEP, malformed preload, bad layout-level import,
etc. — for a one-time ~200-line setup.

## Goal

Three Playwright tests, one assertion shared across all of them: **no CSP
or pageerror console output during a real navigation and click**. The same
harness catches the current bug if it ever recurs.

## Approach

### Packages

- Add dev dep: `@playwright/test`.
- `pnpm dlx playwright install chromium` in CI and locally (one-time).
- Single browser (chromium). Adding Firefox/WebKit is trivial later if
  cross-browser ever matters; smoke doesn't need it.

### Layout

```
playwright.config.ts                 # repo root
tests/
  e2e/
    fixtures.ts                      # test fixture + violation listeners
    global-setup.ts                  # one-time: login, clear must_change_password
    auth.storage.json                # gitignored — saved cookies from global-setup
    smoke.spec.ts                    # the three tests
```

### Server lifecycle

Use Playwright's built-in `webServer` config:

```ts
webServer: {
  command: 'pnpm build && node build/server.js',
  url: 'http://127.0.0.1:4173/login',
  reuseExistingServer: false,
  timeout: 120_000,
  env: {
    PORT: '4173',
    HOST: '127.0.0.1',
    MAW_DATA_DIR: '/tmp/maw-e2e',
    MAW_WORKTREE_ROOT: '/tmp/maw-e2e/worktrees',
    MAW_FIFO_DIR: '/tmp/maw-e2e/fifos',
    MAW_AUTH_LOG_PATH: '/tmp/maw-e2e/auth.log',
    MAW_SESSION_SECRET: 'e2e-secret-not-for-prod',
    MAW_BOOTSTRAP_USERNAME: 'e2e',
    MAW_BOOTSTRAP_PASSWORD: 'e2e-seed-pw-!9x',
    NODE_ENV: 'production'
  }
}
```

- Tests the **bundled production server** (`node build/server.js`) — the
  same artifact the CSP bug manifested in, not `vite dev`.
- Fresh temp data dir each run so the bootstrap path is exercised every
  time. Cleaned up in `globalTeardown`.
- Port 4173 avoids colliding with dev (5173) and prod (3000).
- `Tmux.assertServerRunning` already logs-and-returns on missing socket
  (confirmed in src/lib/server/tmux/TmuxSession.ts:52); no tmux service
  needed for the suite.

### global-setup.ts

One-shot browser script that runs before any test:

1. `page.goto('/login')`, fill `e2e` / `e2e-seed-pw-!9x`, submit.
2. Redirect lands on `/account` because `must_change_password` is set.
3. Fill `current`, `next`, `confirm` with a known test password; submit
   the `?/changePassword` form action.
4. Save `storageState` to `tests/e2e/auth.storage.json`.

The change-password action name (`changePassword`) and field names
(`current` / `next` / `confirm`) come from
src/routes/account/+page.server.ts. /account is allowlisted in
src/hooks.server.ts:32-44 so the action POST goes through cleanly.

Tests load the saved storageState and start already authenticated.

### fixtures.ts

Extends Playwright's `test` with a CSP/error-tracking fixture:

```ts
export const test = base.extend({
  page: async ({ page }, use) => {
    const violations: string[] = [];
    page.on('pageerror', (e) => violations.push(`pageerror: ${e.message}`));
    page.on('console', (m) => {
      const t = m.text();
      if (m.type() === 'error' && /content security policy|refused to execute/i.test(t)) {
        violations.push(`csp: ${t}`);
      }
    });
    await use(page);
    expect(violations, violations.join('\n')).toHaveLength(0);
  }
});
```

Attached once, applied to every test automatically. This is the assertion
that would have caught the current bug.

### smoke.spec.ts — three tests

1. **Dashboard hydrates.** `goto('/')`, wait for `window.__svelte` truthy,
   assert no violations (done in fixture teardown).

2. **User menu opens.** Click `button.user-btn`, expect
   `[role="menu"]` visible and `aria-expanded="true"`.

3. **FAB opens spawn modal.** Click `button.fab`, expect `dialog[open]`
   with an `h2` — we don't assert the i18n string so the test doesn't
   break when locale detection shifts.

All three share `test.use({ storageState: 'tests/e2e/auth.storage.json' })`.

### Scripts + CI

`package.json`:
- `"test:e2e": "playwright test"`
- `"test:e2e:ui": "playwright test --ui"`

`.gitignore`:
- `tests/e2e/auth.storage.json`
- `playwright-report/`
- `test-results/`
- `/tmp/maw-e2e/` stays auto-cleaned by globalTeardown, not gitignored.

`.github/workflows/e2e.yml` (new):
- Triggers on PRs to `main` and `development`.
- Ubuntu runner, Node 24 (matches README).
- Steps: checkout → `pnpm install --frozen-lockfile` →
  `pnpm dlx playwright install --with-deps chromium` → `apt-get install -y
  tmux git` → `pnpm test:e2e`.
- Uploads `playwright-report/` on failure.

No pre-push hook in this pass — keeps the change reviewable and avoids
forcing husky/lefthook on contributors. Revisit once the suite's steady.

## Verification

- `pnpm test:e2e` locally passes against the current (now-fixed) tree.
- **Regression check:** revert the `kit.csp` config in svelte.config.js
  and re-add the hand-rolled CSP in src/hooks.server.ts; `pnpm test:e2e`
  fails — specifically, every test records a "csp: Refused to execute
  inline script" violation. Re-apply the fix; suite passes again.
- CI: push a throwaway branch with the suite, confirm the workflow runs
  on PR and blocks merge on failure.
