# v0.2 — Adapter `createWorktree` flag

> After approval, rename to `v0.2-adapter-create-worktree-flag.md` via `git mv`
> (per `CLAUDE.md` plan convention).

## Context

Every agent spawn currently creates a dedicated git worktree under
`<worktreeRoot>/<agentId>` with a fresh `maw/<agentId>` branch. That is
exactly what coding-agent CLIs (Claude Code, Codex, Gemini) need, but it is
wrong for the `shell.jsonc` smoke adapter — we want to drop into the repo
root on whatever branch it is already on, not a disposable side branch.

Add an adapter-level JSONC flag `createWorktree` (default `true`). When
`false`, spawn skips `git worktree add` and the agent's tmux session cwd
becomes the repo root. Turn it off in `shell.jsonc`; leave the real CLI
adapters alone.

## Approach

Keep the data model unchanged — still insert one `worktrees` row per spawn
(avoids an SQLite table rebuild for a nullable `agents.worktree_id`). For
no-worktree spawns the row points at `repo.path` with `branch =
project.default_branch`, and the agent delete handler learns to skip
`git worktree remove` when the stored path equals the repo path.

Adapter field follows the existing `scrollbackMode` / `historySource`
pattern: Zod schema default → readonly property on `ConfigDrivenAdapter` →
consumer code reads `adapter.createWorktree` (or a small registry helper).

Add to `adapterConfigSchema` (around line 67, next to `scrollbackMode`):

```ts
/**
 * Whether spawning an agent of this kind should create a dedicated git
 * worktree under <worktreeRoot>/<agentId>. Defaults to true — what every
 * real coding-agent CLI wants. Set to false for adapters that should run
 * directly in the repo root on whatever branch is already checked out
 * (e.g. the shell smoke adapter).
```

### 5. `src/lib/server/agents/adapters/AdapterRegistry.ts`

Add a single-purpose helper so the spawn route can branch without
constructing a full adapter instance (cheap, no pattern compilation):

```ts
/** Adapter-declared opt-out of per-agent worktree creation. Defaults true
 *  when the kind is unknown (defensive — caller already validated). */
shouldCreateWorktree(kind: string): boolean {
  const entry = this.entries.get(kind);
  return entry?.config.createWorktree ?? true;
}
```

### 6. `src/routes/agents/new/+page.server.ts`

Branch the spawn flow on the flag. Replace lines ~85–111 with:

```ts
const cfg = getConfig();
const wtm = new WorktreeManager(cfg.worktreeRoot);
const shouldCreate = locals.supervisor.registry.shouldCreateWorktree(role.cli_kind);

let worktreePath: string;
let worktreeBranch: string;

if (shouldCreate) {
  try {
    worktreePath = await wtm.create({
      repoPath: repo.path,
      agentId,
      branch,
      startPoint: project.default_branch
    });
  } catch (err) {
    return fail(400, {
      ...fields,
      error: t(locals.locale, 'spawn.error.worktreeFailed', {
        message: (err as Error).message
      })
    });
  }
  worktreeBranch = branch;
} else {
  worktreePath = repo.path;
  worktreeBranch = project.default_branch;
}

const worktreeId = ulid();
insertWorktree({
  id: worktreeId,
  user_id: locals.user.id,
  repo_id: repo.id,
  path: worktreePath,
  branch: worktreeBranch,
  status: 'active'
});
```

`supervisor.spawn()` already uses `args.worktreePath` as the tmux cwd via
`buildSpawnSpec(...worktreeCwd: args.worktreePath)` → `SpawnSpec.cwd` →
`TmuxSession.newSession({ cwd })` — no changes needed there.

### 7. `src/routes/api/agents/[id]/+server.ts`

Guard `wtm.remove` so we never `git worktree remove` the repo root.
Around line 44, change the inner block to:

```ts
if (worktree && repo) {
  if (worktree.path === repo.path) {
    // No-worktree adapter (e.g. shell smoke) — nothing to remove from disk.
    updateWorktreeStatus(worktree.id, 'removed');
  } else {
    try {
      const wtm = new WorktreeManager(getConfig().worktreeRoot);
      await wtm.remove({ repoPath: repo.path, wtPath: worktree.path, force: true });
      updateWorktreeStatus(worktree.id, 'removed');
    } catch (err) {
      console.error('[delete-agent] worktree remove failed', err);
    }
  }
}
```

## Verification

1. **Type / schema check**
   - `pnpm check` → 0/0.

2. **No-worktree spawn (shell)**
   - Spawn a shell agent from the dashboard.
   - `ls <worktreeRoot>` → no new `<agentId>` directory appears.
   - `git -C <repo> worktree list` → only the root worktree.
   - `git -C <repo> branch` → no new `maw/<agentId>` branch.
   - Tmux session's starting cwd is `<repo.path>` (confirm via
     `tmux display-message -t <session> -p '#{pane_current_path}'`).
   - Terminal modal opens cleanly; basic `ls` / `pwd` in the pane confirms
     we are in the repo root.

3. **Regular spawn (claude-code or any unchanged adapter)**
   - Spawn a claude-code agent.
   - `<worktreeRoot>/<agentId>` exists and is a fresh worktree on
     `maw/<agentId>`.
   - Behavior identical to pre-change.

4. **Delete agent with `removeWorktree=1`**
   - Delete a shell agent: repo root is untouched, worktree row is marked
     `status = 'removed'`, no `[delete-agent] worktree remove failed` logs.
   - Delete a claude-code agent: its dedicated worktree is removed from
     disk as before.

5. **Adapter hot reload**
   - Toggle `createWorktree` in `shell.jsonc`, save, spawn again — the new
     value takes effect on the next spawn (in-flight agents keep their
     snapshot, as with every other adapter field).
