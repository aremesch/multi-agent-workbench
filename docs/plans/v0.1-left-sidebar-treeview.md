# Left Sidebar Treeview + Per-Repo Dashboard

## Context

Today the dashboard (`/`) shows every agent owned by the user in one
gridstack, and archived agents live in a right-side drawer opened from
the hamburger menu. There is no per-repo focus view and no fast way to
see "what's running in repo X" without scanning every card.

This change introduces a persistent, collapsible **left sidebar** with a
treeview of repositories → agents (plus an `Archive` top-level node
grouping archived agents by repo), adds a per-repo dashboard route, and
removes the right-side archive drawer entirely. The hamburger menu is
trimmed to **Settings** (new placeholder route) and **Logout**.

## Design

### Sidebar

- New component `src/lib/client/components/RepoTreeSidebar.svelte`,
  rendered in `src/routes/+layout.svelte` to the left of `<main>`.
- Persistent, collapsible. Two states: expanded (~260px) and collapsed
  (icon-only rail, ~48px). Toggle button in the sidebar header.
- Collapsed/expanded state persisted via existing
  `setUserSetting(userId, 'ui.sidebar.collapsed', json)` mechanism
  (already used for `dashboard.layout.v3` — see
  `src/lib/server/db/queries.ts:608` and `src/lib/shared/dashboard.ts`).
  Hydrated through `+layout.server.ts`.
- Tree structure:
  - **Repositories** (flat list of repos the user owns)
    - Repo node → on click navigates to `/repos/[id]`.
    - Children: active agents (`spawning|running|waiting_input|idle`).
      Click → navigate to `/repos/[id]?agent=<agentId>` (opens the
      repo dashboard with the agent modal pre-opened).
  - **Archive** (top-level)
    - Repo nodes (only repos that have ≥1 archived agent).
      Children: archived agents (`exited|crashed`).
      Click agent → `/repos/[id]?agent=<agentId>` (same URL
      contract; modal works for archived agents too).
- Each agent row shows status dot + name; truncate with title attr.
- No new tree primitive — a small recursive `<ul>`/`<li>` block in the
  sidebar component is enough; no shadcn-svelte tree exists and we
  don't need one.

### Data loading

- Move repo+agent listing for the sidebar into
  `src/routes/+layout.server.ts` so every page has it.
  Reuse:
  - `listAgentCardsForUser(userId, statuses)` —
    `src/lib/server/db/queries.ts:250` — call once with all statuses
    and split client-side, OR call twice (active vs archived). Prefer
    a single call to keep the layout load cheap.
  - Repo names are already joined into `AgentCardRow`; no extra repo
    query is needed for the sidebar (repos with zero agents won't
    appear, which matches the intent — the tree shows what exists).
- Drop `archivedAgents` from `+page.server.ts` (no longer needed by the
  removed drawer); `/` keeps loading active agents only for its grid.

### Per-repo dashboard route

- New route `src/routes/repos/[id]/+page.svelte` and
  `+page.server.ts`.
- `+page.server.ts` loads:
  - The repo (via existing `getRepoById` / similar — verify in
    `queries.ts`; if not present, add a minimal `getRepo(id, userId)`
    helper next to `listReposForProject`).
  - `listAgentCardsForUser(userId, allStatuses)` filtered to this
    repo, OR add `listAgentCardsForRepo(repoId, userId)` mirroring the
    existing query. Prefer a new dedicated query — cleaner and matches
    the existing pattern.
  - Per-repo dashboard layout key:
    `dashboard.layout.repo.<repoId>.v1` (separate from the global
    `dashboard.layout.v3` so each repo has its own gridstack arrangement).
- `+page.svelte` reuses the existing `AgentGrid`, `AgentCard`,
  `Modal`, `AgentTerminalPanel`, and `SpawnAgentForm` components from
  `/`. Effectively a parameterized copy of `src/routes/+page.svelte`.
- Reads `?agent=<id>` from `$page.url.searchParams` on mount; if
  present and matches an agent in the loaded set, sets `openAgent`
  to open the modal. Closing the modal clears the param via
  `goto(..., { replaceState: true, keepFocus: true })`.
- Existing `PUT /api/user/dashboard-layout` accepts a layout key —
  if it's currently hard-coded to `'dashboard.layout.v3'`, extend it
  to accept a key in the request body (validated against an
  allow-list pattern `dashboard.layout(.repo.<uuid>)?.vN`). Verify
  during implementation in
  `src/routes/api/user/dashboard-layout/+server.ts`.

### Layout / hamburger cleanup

- `src/routes/+layout.svelte`:
  - Add sidebar to the left of `<main>` (flex row).
  - Remove `AgentArchiveDrawer` import + render (lines ~5, 94–99).
  - Remove `drawerOpen` state and `openArchive()` handler.
  - Replace hamburger menu items with:
    - **Settings** → navigates to `/settings`.
    - **Logout** → existing logout action.
- Delete `src/lib/client/components/AgentArchiveDrawer.svelte` (no
  remaining consumers).

### Settings route (placeholder)

- New route `src/routes/settings/+page.svelte` with a minimal page
  ("Settings" heading + "coming soon" copy). No server load needed
  yet. Future settings work can fill it in; the menu entry just needs
  a real destination.
