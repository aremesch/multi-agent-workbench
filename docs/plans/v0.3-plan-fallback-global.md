# Fix: Plan not found when `docs/plans` is missing — also surface plans from `~/.claude/plans`

## Context

When a user clicks the agent kebab → **Show Plan**, MAW opens [PlanViewerModal.svelte](src/lib/client/components/PlanViewerModal.svelte) which fetches `/api/agents/:id/plan`. The backend resolves `<worktree>/<plansDirectory>` — defaulting to `docs/plans` per [agentPlans.ts:46](src/lib/server/plans/agentPlans.ts:46) — and lists `*.md` files. If the directory is missing or empty, the modal shows an empty state ("No plan files found in `docs/plans`"), and if a once-listed file vanishes, the render endpoint returns `plan_not_found`.

The trouble: **Claude Code's default plan-output location is `~/.claude/plans/`**, not the worktree. Only projects whose `.claude/settings.json` sets `plansDirectory` get plans inside the worktree. The MAW repo itself does this ([.claude/settings.json:3](.claude/settings.json:3)), but most user projects don't — and on this system, `~/.claude/plans/` already contains 17 plan files generated by Claude Code in plan mode against various repos. MAW is blind to all of them.

**Goal:** make plans discoverable by also listing `~/.claude/plans/`, with each entry labelled by its source. Address the secondary question (auto-create `docs/plans`?) with a recommendation.

## Recommendation on auto-creating `docs/plans` — **No, leave it to the maintainer**

Three reasons:

1. **Pollutes uninvolved projects.** Every spawned agent (claude-code or otherwise) would get an empty `docs/plans/` in its worktree, even when the user never asked for project-local plans. Some projects have their own conventions (`docs/design/`, `notes/`, none at all).
2. **Doesn't actually solve the bug.** Claude Code only writes plans into the worktree when `.claude/settings.json#plansDirectory` is set. Creating an empty `docs/plans/` doesn't change Claude's output location — plans still go to `~/.claude/plans/`.
3. **Convention is the maintainer's call.** Adding `plansDirectory` to `.claude/settings.json` (and committing the dir) is one line of project setup; doing it implicitly steals that decision.

The fallback to `~/.claude/plans/` makes project-side setup optional rather than required, which is the right ergonomics.

## Approach: always merge local + global, sorted by mtime, source-tagged

Two sources, always both consulted, returned as a single list sorted by `modifiedMs` desc:

| Source tag | Path | Filter |
|---|---|---|
| `local` | `<worktree>/<plansDirectory>` | git-touched on this branch since `base_sha` (existing) |
| `global` | `<os.homedir()>/.claude/plans` | `mtime ≥ agent.created_at − 60_000` (skew buffer) |

**Why a 60s skew buffer:** Claude Code writes the plan file at the moment plan-mode activates, which can be a fraction of a second before MAW records `agent.created_at`. Without slack we lose plans authored by the just-spawned agent. 60s is generous enough to cover wall-clock jitter and small clock corrections, narrow enough to exclude older sessions.

**Why mtime (heuristic, not exact):** there is no authoritative agent↔global-plan link short of parsing Claude Code's stdout for the plan path it announces. That's a much bigger change and lives in [claudeHooks.ts](src/lib/server/agents/claudeHooks.ts). For this fix we accept the heuristic and document the imprecision in code.

## Implementation

### 1. `src/lib/server/plans/agentPlans.ts` — core changes

- Add `import { homedir } from 'node:os';` and `import { realpath } from 'node:fs/promises';`.
- Add a constant + helper:
  ```ts
  const GLOBAL_DISPLAY = '~/.claude/plans';
  /** Claude Code's default plan-output dir when no project-level
   *  plansDirectory is configured. Single-user app, same UID as MAW. */
  function globalPlansDir(): string {
    return resolve(homedir(), '.claude', 'plans');
  }
  ```
- Extend `PlanFileSummary` with `source: 'local' | 'global'`.
- Extract a small helper `listMdFromDir(dirAbs)` that does `readdir` → filter by `SAFE_FILENAME_RE` → parallel `stat` and returns `{ name, modifiedMs, sizeBytes }[]`. Returns `[]` cleanly on `ENOENT` (so a missing `~/.claude/plans` is fine for fresh users). Used by both branches.
- New `listGlobalPlans(agentCreatedAtMs: number): Promise<PlanFileSummary[]>`:
  - Calls `listMdFromDir(globalPlansDir())`.
  - Filters `modifiedMs >= agentCreatedAtMs - 60_000`.
  - Tags each with `source: 'global'`.
  - Code-comments the heuristic and the −60s skew rationale.
- Change `listAgentPlans(worktreePath, plansDir, baseSha, agentCreatedAtMs)`:
  - Run local listing (existing logic, refactored to use `listMdFromDir` + git filter) and `listGlobalPlans` in parallel.
  - Tag local results with `source: 'local'`.
  - Concatenate, sort by `modifiedMs` desc (single key — ties keep insertion order, deterministic enough; document the choice).
- Change `renderAgentPlan(worktreePath, plansDir, filename, source)`:
  - `dirAbs = source === 'global' ? globalPlansDir() : resolve(worktreePath, plansDir)`.
  - Resolve to canonical paths via `realpath(dirAbs)` then build `fileAbs = resolve(realDir, filename)` and re-check `fileAbs.startsWith(realDir + sep)` — closes the symlink TOCTOU on `~/.claude` flagged by review.
  - Same `SAFE_FILENAME_RE` and `invalid_filename` throw on bad input.
- Export a tiny `displayDir(source: 'local' | 'global', plansDir: string)` returning `plansDir` or `GLOBAL_DISPLAY` for the empty-state copy.

### 2. `src/routes/api/agents/[id]/plan/+server.ts` — wire it through

- Pass `agent.created_at` into `listAgentPlans` (4th arg).
- For the `?file=` branch, read `?source=` (default `'local'`), validate it is exactly `'local'` or `'global'`, return `400 invalid_source` otherwise. Validate **before** calling `resolvePlansDir` for the global case (no need to read settings.json when we're rendering a global file — minor perf, also belt-and-braces).
- List response shape:
  ```ts
  { dir: string; globalDir: string; files: PlanFileSummary[] }
  ```
  `globalDir` is the user-friendly `~/.claude/plans` string (from `displayDir`), used by the empty-state copy.

### 3. `src/lib/client/components/PlanViewerModal.svelte` — UI changes

- Add `source: 'local' | 'global'` to the local `FileSummary` type.
- `loadFile` appends `&source=${source}` to the render fetch.
- Render-time labelling:
  - In the switcher dropdown: `{file.name} · {fmtTime}{file.source === 'global' ? ' · global' : ''}`.
  - Modal title: same tag when current is global.
- Empty-state uses a new i18n key that mentions both locations, fed from `body.dir` and `body.globalDir`.
- Auto-loaded first file uses its own `source` field (since the merged list might have global at index 0).

### 4. i18n (`src/lib/i18n/{en,de,es,fr}.ts`)

Update one key, add one key, in all four locales:

- **Update** `plan.modal.empty`: `'No plan files found in {dir} or {globalDir}'` (en) — translators adapt the wording.
- **Add** `plan.modal.sourceGlobal`: short label `'global'` (en) used as a suffix in the switcher.

For non-English locales, follow the existing tone in each file. No new copy keys beyond these two.

### 5. Tests

#### `src/lib/server/plans/agentPlans.test.ts` (existing — extend)

Mock `node:os` with `vi.mock('node:os', () => ({ homedir: () => '/home/test' }))`. Add to the existing `node:fs/promises` mock: `realpath: (p) => realpathMock(p)` (default identity).

New cases (alongside existing):
- Local has 2 files (touched), global has 1 newer + 1 older than `agentCreatedAtMs − 60s`: returns 3 files (local both, global newer one), sorted by mtime desc, `source` tags correct.
- Local empty, global has 1 newer: returns the global one tagged `source: 'global'`.
- Local empty, global only has files **older than** `agentCreatedAtMs − 60s`: returns `[]`.
- Global dir doesn't exist (`readdir` rejects ENOENT): returns local-only without throwing. (Fresh-user case.)
- Skew buffer: file at exactly `agentCreatedAtMs − 60_000` is included; at `agentCreatedAtMs − 60_001` is excluded.
- `renderAgentPlan(_, _, name, 'global')` reads from `/home/test/.claude/plans/<name>.md` and rejects path traversal the same as `'local'`.
- `renderAgentPlan` symlink defence: `realpath` returns a path *outside* the dir → throws `invalid_filename` (TOCTOU defence).

#### `src/routes/api/agents/[id]/plan/server.test.ts` (existing — extend)
