# Comprehensive vitest unit-test suite

## Context

MAW currently has **zero unit tests**. The only automated coverage is a
three-case Playwright hydration smoke suite (`tests/e2e/smoke.spec.ts`)
that runs against the bundled prod server. Vitest is already a
devDependency and `pnpm test` is wired up (see
[package.json:14-16](package.json:14)), but
[vite.config.ts:61-63](vite.config.ts:61) declares only a file include
pattern ‚Äî no environment, no setup, no coverage.

That is a lot of load-bearing logic running untested: argon2 session
lifecycle, CSRF double-submit, migration runner, slug generation,
adapter state machines, template substitution, ANSI/SGR parsing,
Claude-JSONL history rendering, WS protocol v3 dispatch, tmux
capture dedup (`collapseRepeatingTailBlocks`), push-notification
filters, and every zod schema we parse untrusted input with. Each has
already regressed at least once (hydration CSP incident, scrollback
ghosting, worktree self-heal edge cases).

The goal is a **phased** unit-test suite that (a) makes
pure/near-pure logic unbreakable, (b) gives future refactors a safety
net before touching tmux/supervisor/WS code, and (c) lands a coverage
gate in CI so the next regression is caught on the PR, not in
production.

## Non-goals

- **Do not** duplicate Playwright hydration smoke coverage. UI
  integration stays in `tests/e2e/`.
- **Do not** touch `src/lib/components/ui/` (vendored shadcn-svelte ‚Äî
  see CLAUDE.md rule).
- **Do not** write integration tests that spawn real tmux, real git,
  or real child processes. All such seams are mocked at the
  `execa`/`node:fs`/`WebSocket` boundary.
- **Do not** chase 100% coverage; see targets in Phase 10.

## Test infrastructure

Vitest today has a single flat config. It needs to host two envs in
one run (server modules import `node:fs`, `node:crypto`,
`better-sqlite3`; client modules use `window`/`document`).

## Phase 2 ‚Äî DB layer ‚úÖ

Use the `openMemoryDb()` helper from `tests/unit/helpers/db.ts`.
Queries tests share one handle per file (state is wiped between
cases via `clearAllTables`) so `queries.ts`'s prepared-statement
cache stays valid; migration-runner tests use a fresh `:memory:` DB
per case via `vi.mock('./index.js', ...)`.

**Landed:** migrate.test.ts (10 cases) + queries.test.ts (55
cases) + helper now real. Coverage 9.39 ‚Üí **15.96 lines / 78.76
branches / 39.20 functions / 15.96 statements**; `src/lib/server/db`
alone reached ~96% line coverage. Threshold ratcheted to
15/78/38/15. One real bug caught by the test: `pruneTerminalLogByBytes`
iterated a SELECT while a DELETE ran on the same connection, which
better-sqlite3 rejects ("connection busy"); fixed by materializing
the id/size list with `.all()` before the DELETE loop.

### Migration runner ‚Äî `src/lib/server/db/migrate.ts`

- Applies all `migrations/*.sql` in file-name order on a fresh DB.
- Idempotent: second `runMigrations()` is a no-op.
- Records SHA-256 hash in `__drizzle_migrations` (or equivalent
  tracking table ‚Äî confirm exact name from the file).
- Hash-mismatch detection (same filename, altered content) is flagged.
- A failing migration leaves the tracking table consistent (rolled
  back inside its tx).
- FK-pragma dance does not permanently disable FKs.

### Query helpers ‚Äî `src/lib/server/db/queries.ts`

Smoke each CRUD pair (insert ‚Üí get ‚Üí list ‚Üí update ‚Üí delete) against
`openMemoryDb()`. Priority helpers (from CLAUDE.md hot paths):

- User: `getUserByUsername`, `insertUser`.
- Session: `insertSession`, `getSessionById`, `deleteSession`, expiry
  boundary.
- Project/Repo/Role: CRUD + the `listReposWithProjectForUser`
  left-joins that power the sidebar.
- Agent: `insertAgent`, `getAgent`, `listLiveAgents`,
  `updateAgentStatus` state transitions, `cli_session_id` write/read.
- Terminal log: `insertTerminalChunk`, log size/retention query if any.
- Push: `insertPushSub`, `listPushSubsForUser`,
  `deletePushSubByEndpoint`.
- Auth events: `insertAuthEvent` + listing for recent-login UI.

## Phase 3 ‚Äî Adapter engine ‚úÖ

The adapter layer is where untrusted user-editable JSONC meets a
state-machine that decides when to fire push alerts. **Highest
blast-radius-per-bug tier.**

**Landed:** adapter.config.schema (24 cases) +
ConfigDrivenAdapter (36 cases) + AdapterRegistry (13 cases) =
**73 new cases**. Hot-reload via chokidar deliberately left for a
later pass ‚Äî the unit tests cover `loadFile` semantics, which is
what `startWatching` delegates to. Overall coverage
15.96 ‚Üí **18.87 lines / 82.29 branches / 44.83 functions /
18.87 statements**; `src/lib/server/agents/adapters` alone at
~91% line / ~94% branch. Threshold ratcheted to 18/82/44/18.

### `adapter.config.schema.ts` (zod)

- Every required field missing ‚Üí specific error path.
- Default application (`scrollbackMode` defaults to `'visible'`,
  `idleDetection.inactivityMs` default, etc.).
- Invalid regex strings rejected at parse time (with file path in
  error message for operator UX).
- `historySource.kind: 'claude-jsonl'` accepted; unknown kind
  rejected.
- `input.encode` / `input.answerPrompt` union variants validate.

### `ConfigDrivenAdapter` ‚Äî [src/lib/server/agents/adapters/ConfigDrivenAdapter.ts](src/lib/server/agents/adapters/ConfigDrivenAdapter.ts)

Feed synthesized byte chunks through `ingest()` and assert on emitted
events + `state()` transitions.

- State machine: `BOOTING ‚Üí READY ‚Üí WORKING ‚Üí WAITING_PROMPT ‚Üí READY`
  round-trip driven by configured patterns.
- ANSI stripped before pattern matching (feed a chunk with colour
  codes, assert pattern still fires).
- Tail buffer sliding: feed > 8 KB, ensure older content drops out and
  patterns only match recent tail.
- `buildSpawnSpec` template substitution:
  `{{agent.cliSessionId}}`, `{{agent.worktreePath}}`,
  `{{role.systemPrompt}}`, `{{env.FOO}}` ‚Äî including missing-var
  fallback behaviour.
- `firedMatches` dedup: same alert pattern in two consecutive chunks
  fires once; different patterns both fire.
- `isIdleWaiting()` dual trigger: cursor-at-prompt regex match and
  inactivity-timeout path (fake timers).

### `AdapterRegistry` ‚Äî same dir

- `loadAll()` reads fixture dir, returns `{ adapters, errors }`.
- Two JSONC files declaring the same `kind` ‚Üí duplicate error, both
  listed.
- Malformed JSONC ‚Üí parse error with file path.
- Schema-invalid config ‚Üí zod error with file path.
- `create(kind)` returns a fresh `ConfigDrivenAdapter`; unknown kind
  throws.
- Hot-reload: mock chokidar, fire `change` on a fixture, assert
  registry swap + debounce (fake timers).

## Phase 5 ‚Äî Auth

### Password ‚Äî `src/lib/server/auth/password.ts`

- `hashPassword` + `verifyPassword` round-trip (single happy case is
  enough given argon2 is a library).
- `verifyPassword` rejects on bad hash prefix without throwing.
- Parameter tampering: wrong password ‚Üí `false`.

### Session ‚Äî `src/lib/server/auth/session.ts`

- `createSession` returns a ULID-shaped id, expiry = now + TTL.
- `resolveSession` returns user on fresh session, `null` past expiry.
- Cookie set/clear round-trip through a minimal `RequestEvent` fake.

### CSRF ‚Äî `src/lib/server/auth/csrf.ts`

- `ensureCsrfCookie` issues on empty, preserves on present.
- `verifyCsrf` timing-safe compare: equal ‚Üí true, differing length ‚Üí
  false without throwing.
- Double-submit header mismatch ‚Üí false.

### Rate limiter ‚Äî `src/lib/server/auth/rateLimit.ts`

- In-memory bucket allows `limit` hits, blocks the `limit+1`-th.
- Window rolls: after `windowMs`, bucket reopens.
- Different keys are independent.
- Cleanup pass drops stale buckets (fake timers).

### Auth log ‚Äî `src/lib/server/auth/authLog.ts`

- Dual-write: inserts DB row and appends file line.
- Never throws: DB insert error is swallowed + logged; file error
  ditto.
- Sanitization: password-ish fields are never serialized.

## Phase 10 ‚Äî CI + coverage gate

### New file ‚Äî `.github/workflows/unit.yml`

Mirror `.github/workflows/e2e.yml` (per CLAUDE.md Playwright plan):

```yaml
name: Unit tests
on:
  pull_request:
    branches: [main, development]
  push:
    branches: [main, development]
jobs:
  vitest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: pnpm }
      - run: pnpm install --frozen-lockfile
      - run: pnpm test -- --coverage
      - uses: actions/upload-artifact@v4
        if: always()
        with: { name: coverage, path: coverage/ }
```

### Coverage thresholds

Start at **70% lines / 60% branches / 70% functions / 70%
statements** on non-vendored `src/`. Raise in 5-point increments as

## Verification

End-to-end proof that the suite works and the gate bites:

1. `pnpm install` ‚Äî new devDeps resolve.
2. `pnpm test` ‚Äî every phase's tests pass locally on a fresh checkout.
   Both projects (`server` node env + `client` jsdom env) report in
   the run output.
3. `pnpm test -- --coverage` ‚Äî writes `coverage/` with lcov + HTML;
   thresholds met.
4. Deliberately break a covered function (e.g. change
   `slugifyTitle` to skip the length cap) ‚Üí `pnpm test` fails with a
   clear message pointing at the broken case. Revert.
5. `pnpm check` ‚Äî still 0/0; no new type errors introduced by
   fixtures or helpers.
6. `pnpm test:e2e` ‚Äî Playwright smoke suite continues to pass
   (regression guard: projects split didn't break the existing
   pipeline).
7. Open a draft PR ‚Äî `unit.yml` workflow runs, reports green, uploads
   coverage artifact.
8. Drop coverage below threshold in the draft PR (remove a test
   temporarily) ‚Üí CI fails on the threshold, not on any individual
   test. Restore before merge.
