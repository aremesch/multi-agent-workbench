# Plan: revert honest-action-status, migrate auth to better-auth

## Context

Two coupled changes:

1. **Revert** the `honestActionStatus` hook landed in commit
   [`d7d96ce`](../../). The HTTP-status-rewriter for SvelteKit's JSON
   form-action path goes away with the rest of the custom auth — there's
   no point preserving a wrapper around code we're replacing.

2. **Replace** the hand-rolled auth (cookie sessions in
   [`src/lib/server/auth/session.ts`](../../src/lib/server/auth/session.ts) +
   argon2id in [`src/lib/server/auth/password.ts`](../../src/lib/server/auth/password.ts) +
   custom rate limiter + opaque session ULIDs in
   [`sessions`](../../migrations/001_init.sql) +
   [`users.password_hash`](../../migrations/001_init.sql)) with
   [better-auth](https://www.better-auth.com/) — a maintained library
   that owns sign-in / sign-out / password-change / session storage.

Why: the custom auth is fine but every change to it (CSRF, rate limiting,
the wire-status fix in `d7d96ce`) is a hand-rolled patch on a hand-rolled
foundation. Outsourcing to a maintained library lets future auth features
(2FA, OAuth, account recovery, …) drop in without re-engineering each
time. Functional surface stays identical for v0.2: **single user, email
sign-in, password change, no signup, no user-management UI**.

## Scope (confirmed)

- **Login identifier:** email. Existing `admin` user becomes `admin@maw.local`.
- **Password:** keep existing argon2id hash. Plug `@node-rs/argon2` in as
  better-auth's custom `password.hash` / `password.verify`; copy
  `users.password_hash` into `account.password` at migration time.
- **fail2ban:** unchanged. The new login/logout/change-password code paths
  still emit the existing `<iso8601> <event> user=<u> ip=<ip> ua="<ua>"`
  lines via [`logAuth()`](../../src/lib/server/auth/authLog.ts).
- **Rate limiter:** use better-auth's built-in (configured at 10/60s on the
  credential sign-in path). Drop the custom
  [`rateLimit.ts`](../../src/lib/server/auth/rateLimit.ts) and its tests.

## Step 3 — Migration `007_better_auth.sql`

Better-auth requires four tables (`user`, `session`, `account`,
`verification`) with its own schema. We keep the existing `users` /
`sessions` snake-case tables for now — `users` is the FK target for ~30
domain tables (`projects.user_id`, `repos.user_id`, `agents.user_id`,
etc.) and ripping out those FKs is a much larger blast radius than this
plan's "swap auth, keep functionality" goal. The two tables coexist; the
migration keeps `users.id` and `user.id` byte-for-byte equal so domain FKs
keep resolving.

```sql
-- Create better-auth's canonical tables (column names per
-- https://www.better-auth.com/docs/concepts/database).
CREATE TABLE user (
  id             TEXT PRIMARY KEY,
  name           TEXT NOT NULL,
  email          TEXT NOT NULL UNIQUE,
  emailVerified  INTEGER NOT NULL DEFAULT 0,
  image          TEXT,
  createdAt      INTEGER NOT NULL,
  updatedAt      INTEGER NOT NULL
);

CREATE TABLE session (
  id         TEXT PRIMARY KEY,
  userId     TEXT NOT NULL REFERENCES user(id) ON DELETE CASCADE,
  token      TEXT NOT NULL UNIQUE,
  expiresAt  INTEGER NOT NULL,
  ipAddress  TEXT,
  userAgent  TEXT,
  createdAt  INTEGER NOT NULL,
  updatedAt  INTEGER NOT NULL
);
CREATE INDEX idx_session_user ON session(userId);

CREATE TABLE account (
  id                     TEXT PRIMARY KEY,
  userId                 TEXT NOT NULL REFERENCES user(id) ON DELETE CASCADE,
  accountId              TEXT NOT NULL,
  providerId             TEXT NOT NULL,
  accessToken            TEXT,
  refreshToken           TEXT,
  accessTokenExpiresAt   INTEGER,
  refreshTokenExpiresAt  INTEGER,
  scope                  TEXT,
  idToken                TEXT,
  password               TEXT,
  createdAt              INTEGER NOT NULL,
  updatedAt              INTEGER NOT NULL
);
CREATE INDEX idx_account_user ON account(userId);

CREATE TABLE verification (
  id          TEXT PRIMARY KEY,
  identifier  TEXT NOT NULL,
  value       TEXT NOT NULL,
  expiresAt   INTEGER NOT NULL,
  createdAt   INTEGER NOT NULL,
  updatedAt   INTEGER NOT NULL
);

-- One-shot copy of the existing single user into better-auth's tables.
-- Email is synthesized as `<username>@maw.local`. Same id ensures domain
-- FKs (projects.user_id, repos.user_id, agents.user_id, …) keep resolving.
INSERT INTO user (id, name, email, emailVerified, createdAt, updatedAt)
SELECT
  id,
  username,
  username || '@maw.local',
  1,
  created_at,
  updated_at
FROM users
WHERE NOT EXISTS (SELECT 1 FROM user WHERE user.id = users.id);

-- Copy the argon2 hash into account.password under the `credential` provider.
-- accountId is the user.id by better-auth convention for credential accounts.
INSERT INTO account (id, userId, accountId, providerId, password, createdAt, updatedAt)
SELECT
  id || '-cred',
  id,
  id,
  'credential',
  password_hash,
  created_at,
  updated_at
FROM users
WHERE NOT EXISTS (
  SELECT 1 FROM account WHERE account.userId = users.id AND account.providerId = 'credential'
);

-- Drop the legacy `sessions` table — better-auth owns sessions now.
-- Existing rows are throwaway (login state); users will sign in again.
DROP INDEX IF EXISTS idx_sessions_user;
DROP INDEX IF EXISTS idx_sessions_expires;
DROP TABLE IF EXISTS sessions;
```

`users.password_hash` and `users.must_change_password` stay on the legacy
table for now (read by nothing after this commit — drop in a follow-up).

## Step 4 — `src/lib/server/auth/betterAuth.ts` (new)

Single source of truth for the better-auth instance. Pulls the same
`getDb()` handle the rest of the app uses, plugs argon2id in as the hasher,
disables signup, and configures the rate limit.

```ts
import { betterAuth } from 'better-auth';
import { sveltekitCookies } from 'better-auth/svelte-kit';
import { getRequestEvent } from '$app/server';
import { hash, verify } from '@node-rs/argon2';
import { getDb } from '../db/index.js';
import { getConfig } from '../config.js';

const argon2Options = { memoryCost: 19456, timeCost: 2, outputLen: 32, parallelism: 1 };

export const auth = betterAuth({
  database: getDb(),                       // existing better-sqlite3 handle
  baseURL: getConfig().publicOrigin || undefined,
  secret: process.env.BETTER_AUTH_SECRET,
  emailAndPassword: {
    enabled: true,
    disableSignUp: true,                   // single-user — no public signup
    minPasswordLength: 8,
    password: {
      hash: (plain) => hash(plain, argon2Options),
      verify: ({ hash: stored, password }) => verify(stored, password)
    }
  },
  rateLimit: {
    enabled: true,
    window: 60,
    max: 10                                // matches old MAW_LOGIN_RATE_LIMIT default
  },
  session: {
    cookieName: 'maw_session',             // keep cookie name for back-compat
    expiresIn: getConfig().sessionTtlSeconds,
    cookieOptions: { sameSite: 'strict', secure: !getConfig().isDev }
  },
  plugins: [sveltekitCookies(getRequestEvent)]
});
```

## Step 6 — Rewrite `src/hooks.server.ts`

The whole file is replaced. No `sequence`, no `honestActionStatus`, no
custom `resolveSession`. After bootstrap, ask better-auth for the session
on every request and adapt the legacy shape so existing call sites
(`event.locals.user.id`, `.username`, etc.) keep working.

```ts
import { redirect, type Handle } from '@sveltejs/kit';
import { bootstrap, getSupervisor } from '$lib/server/bootstrap';
import { auth } from '$lib/server/auth/betterAuth';
import { ensureCsrfCookie } from '$lib/server/auth/csrf';
import { getMustChangePasswordById } from '$lib/server/db/queries';
// theme + locale imports unchanged

await bootstrap();

export const handle: Handle = async ({ event, resolve }) => {
  const sess = await auth.api.getSession({ headers: event.request.headers });
  // Adapt to the existing UserRow shape so call sites don't change.
  // username = email's local-part for display.
  event.locals.user = sess
    ? { id: sess.user.id, username: sess.user.email.split('@')[0],
        must_change_password: getMustChangePasswordById(sess.user.id) ? 1 : 0 }
    : null;
  event.locals.session = sess?.session ?? null;
  event.locals.supervisor = getSupervisor();

  ensureCsrfCookie(event.cookies);

  // must_change_password redirect — unchanged behaviour.
  if (event.locals.user?.must_change_password) {
    const p = event.url.pathname;
    const allow = p === '/account' || p === '/login' || p === '/logout'
      || p.startsWith('/api/auth/') || p.startsWith('/_app/') || p.startsWith('/static/')
      || p === '/service-worker.js' || p === '/favicon.ico' || p === '/manifest.webmanifest';
    if (!allow) throw redirect(303, '/account');
  }

  // theme + locale + transformPageChunk + security headers — unchanged
  return resolve(event, { transformPageChunk: /* unchanged */ });
};
```

`UserRow` type in [`src/lib/server/db/types.ts`](../../src/lib/server/db/types.ts)
stays the same shape — we just stop populating `password_hash` (nothing
reads it after this commit).

## Step 7 — Rewrite the auth routes

### [`src/routes/login/+page.server.ts`](../../src/routes/login/+page.server.ts)

Form swaps `username` → `email`. Action wraps `auth.api.signInEmail` and
emits the same `logAuth(...)` lines so fail2ban regex matches verbatim.

```ts
import { fail, redirect } from '@sveltejs/kit';
import { APIError } from 'better-auth/api';
import { auth } from '$lib/server/auth/betterAuth';
import { logAuth } from '$lib/server/auth/authLog';
import { clientIp } from '$lib/server/net/clientIp';
import { t } from '$lib/i18n';
import { getMustChangePasswordById } from '$lib/server/db/queries';

export const load = async ({ locals }) => {
  if (locals.user) throw redirect(303, '/');
  return {};
};

export const actions = {
  login: async (event) => {
    const { request, locals } = event;
    const ip = clientIp(event);
    const ua = request.headers.get('user-agent');
    const form = await request.formData();
    const email = String(form.get('email') ?? '').trim();
    const password = String(form.get('password') ?? '');
    if (!email || !password) {
      return fail(400, { email, error: t(locals.locale, 'login.error.required') });
    }
    try {
      const { user } = await auth.api.signInEmail({
        body: { email, password },
        headers: request.headers,
        asResponse: false
      });
      logAuth('login_ok', { userId: user.id, username: email, ip, userAgent: ua });
      if (getMustChangePasswordById(user.id)) throw redirect(303, '/account');
      throw redirect(303, '/');
    } catch (e) {
      if (e instanceof APIError && e.status === 'TOO_MANY_REQUESTS') {
        logAuth('rate_limited', { username: email, ip, userAgent: ua });
        return fail(429, { email, error: t(locals.locale, 'login.error.rateLimited') });
      }
      if (e instanceof APIError /* INVALID_EMAIL_OR_PASSWORD, etc. */) {
        logAuth('login_fail', { username: email, ip, userAgent: ua, detail: e.code });
        return fail(401, { email, error: t(locals.locale, 'login.error.invalid') });
      }
      throw e;
    }
  },
  logout: async (event) => {
    const ip = clientIp(event);
    const ua = event.request.headers.get('user-agent');
    const userId = event.locals.user?.id ?? null;
    const username = event.locals.user?.username ?? null;
    await auth.api.signOut({ headers: event.request.headers });
    logAuth('session_revoked', { userId, username, ip, userAgent: ua });
    throw redirect(303, '/login');
  }
};
```

## Implementation notes (deviations from the original spec)

What landed differs from the original sketch in four places — recorded
here for future readers:

- **Cookie forwarding without `sveltekitCookies`.** The plan called for
  `plugins: [sveltekitCookies(getRequestEvent)]`. That import pulls
  `$app/server`, a SvelteKit-only virtual alias. The prod server is
  bundled twice (vite for SvelteKit's chunks, esbuild for `build/server.js`)
  and only the vite path resolves the alias; the esbuild path crashes at
  load. `betterAuth.ts` ships the auth instance without that plugin, and
  `/login`, `/logout`, `/account` actions call `auth.api.*({ asResponse:
  true })` and pass the resulting Response through a small
  `forwardCookiesFromResponse` helper that uses
  `parseSetCookieHeader` + `toCookieOptions` from `better-auth/cookies`.
  Net behaviour is identical (Set-Cookie still lands on `event.cookies`),
  no SvelteKit alias required.
- **In-memory `checkRate` retained.** The plan said better-auth's built-in
  rate limiter would replace `src/lib/server/auth/rateLimit.ts`. In
  practice better-auth's limiter only fires for its HTTP routes
  (`/api/auth/*`); the `/login` form action calls `auth.api.signInEmail`
  programmatically, bypassing the middleware. The custom limiter stays
  in front of the action so password sprays still trigger
  `rate_limited` lines for fail2ban. Configured via the existing
  `MAW_LOGIN_RATE_LIMIT` env (default 10/60s).
- **Two-stage build externalization.** Vite's `ssr.external` keeps
  better-auth out of SvelteKit's SSR chunks (so the precompiled
  better-auth code resolves to its own zod@4 at runtime); esbuild's
  `external` in `scripts/bundle-server.mjs` does the same for
  `build/server.js` (the app's top-level `zod` is still v3 for the
  adapter schemas). Without both, the server crashes at boot with
  `z.coerce.boolean(...).meta is not a function`.
- **`pwchange_fail` path requires `resp.ok` check.** With
  `asResponse: true`, better-auth no longer throws `APIError` on a wrong
  current password — it returns a 400 Response with `{ code:
  'INVALID_PASSWORD' }`. A first-pass `/account` action relied on the
  exception path and silently logged `pwchange_ok` for failed attempts;
  the shipped version inspects `resp.ok` and parses the body's `code`
  explicitly. Same shape applies to `/login` (handled by the existing
  `data.user?.id` check plus an explicit `!resp.ok` branch).
