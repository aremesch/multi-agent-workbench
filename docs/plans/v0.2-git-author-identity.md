# Configurable Git Author Identity

## Context

MAW runs on an LXC container as Unix user `maw`. Commits made *by a spawned agent* currently pick up `GIT_COMMITTER_*` from the env MAW injects (`MAW-Agent-<id>` / `<id>@maw.local`, load-bearing for attribution), but **`GIT_AUTHOR_*` is never set** ‚Äî so author falls back to the system git config of the `maw` user. That ops-user identity leaks into commit history and, worse, into anything the user pushes to a public GitHub repo.

MAW already has a proper login user (`locals.user`). The login user's identity should drive the git author. A secondary leak lives in `WorktreeManager.MAW_COMMIT_IDENTITY` ‚Äî hardcoded `Multi-Agent Workbench / maw@localhost` used for admin commits (empty-repo init, unborn-repo seed) that also land in the user's repo history.

Desired outcome: each user can set `git.authorName` + `git.authorEmail` in `/settings`; those drive both agent commits and MAW's own admin commits; the committer side is untouched so `commitSnapshot.ts` attribution keeps working.

---

## Storage: `user_settings` (no migration)

Two dot-namespaced keys: `git.authorName`, `git.authorEmail`, one string per key. Matches the prior art (`theme`, `locale`, `ui.sidebar.collapsed`, `push.prefs`, `spawn.defaults.*`). Rationale:
- `users` is deliberately authn-only (`username`, `password_hash`, ‚Ä¶). Two optional strings don't belong there.
- Prior art is one-value-per-key (see `theme`, `locale`) ‚Äî matches better than a single `git.author` JSON blob for independent field save/load.
- Zero schema changes.

Helpers (new module `src/lib/server/user/gitIdentity.ts`):
- `getGitIdentity(userId, username): GitIdentity` ‚Äî reads the two keys; **falls back** to `{ name: username, email: <username>@maw.local }` when unset so spawn never fails.
- `setGitIdentity(userId, { name, email })` ‚Äî validates, upserts via existing `setUserSetting`.
- `hasGitIdentity(userId): boolean` ‚Äî drives the "configure your git identity" banner.

Validation (server-side):
- `name`: 1..100 chars, no leading/trailing whitespace, reject `<`/`>` (git itself rejects them).
- `email`: matches `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`, ‚â§254 chars, no `<`/`>`.

## Public-GitHub safety guidance (for the UI helper text)

GitHub provides a privacy-protected email per user in the form:

```
<user-id>+<username>@users.noreply.github.com
```

(e.g. `12345678+alice@users.noreply.github.com`) ‚Äî the exact value is shown at **GitHub ‚Üí Settings ‚Üí Emails** under "Keep my email addresses private". Using it for the git author email:

- Still produces clickable author avatars/links on github.com (GitHub maps the noreply back to the account).
- Keeps the user's real inbox out of the public commit history, which is immutable once pushed.
- Pairs with GitHub's "**Block command line pushes that expose my email**" checkbox as a belt-and-suspenders safeguard worth enabling.

The settings form should surface this as helper text under the email field, e.g.:

> Tip: For public repositories, use your GitHub privacy email (`<id>+<username>@users.noreply.github.com`) ‚Äî find it under GitHub ‚Üí Settings ‚Üí Emails.

Any real email the user enters here will end up in the public commit log forever, so linking them to the option is the right default.

**Preparation work the user should do (outside code):**
1. Visit `https://github.com/settings/emails` while logged in as the GitHub account they'll push from.
2. Copy the `Keep my email addresses private` value and paste it into MAW's settings.
3. (Optional) Enable "Block command line pushes that expose my email" on GitHub as a guardrail.

No MAW-side prep is needed ‚Äî the SSH/HTTPS push path already uses whatever credentials the agent's shell is configured with.

## Gotchas

- **Reattach-on-boot keeps the old env.** `AgentSupervisor.init()` does *not* respawn the tmux session, so agents that were spawned before the user configured identity keep their original fallback author. Changing identity does not retroactively alter any live session. Document this in the settings helper text: *"Applies to agents spawned after saving."*
- **`GIT_AUTHOR_DATE`** ‚Äî don't set. Git defaults author date to commit time, which is correct.
- **Admin commits from `initEmpty` / `ensureDefaultBranch`** are attributed to the user who attached the repo. Acceptable ‚Äî it's their action ‚Äî but worth one line in the commit message of this change.
- **`commitSnapshot.ts` attribution** keys on `committer_email` only, and `agent_commits` stores author separately ‚Äî changing author is safe for the read side.
- **No new user_settings schema validation layer** ‚Äî the value is JSON text per existing contract. Encode with `JSON.stringify(name)` / `JSON.stringify(email)` (scalar strings), decode with `JSON.parse`, matching existing setters.

1. `pnpm check` ‚Äî typecheck clean after refactor (new param on `initEmpty` / `ensureDefaultBranch` is a compile-time contract change, covering all callers).
2. `pnpm test` ‚Äî add unit tests for `gitIdentity.ts` (fallback, validation round-trip). Update `WorktreeManager.test.ts` if its tests exercise `initEmpty` / `ensureDefaultBranch` signatures.
3. Manual end-to-end (via `ssh maw@maw` or against `https://maw.remesch.at/`):
   1. **Unset-identity path**: leave `git.author*` unset, spawn a `shell` agent, in the pane run `git commit --allow-empty -m t && git log -1 --pretty=format:'%an <%ae> | %cn <%ce>'`. Expect `author = <username> <<username>@maw.local>`, `committer = MAW-Agent-<id> <<id>@maw.local>`.
   2. **Configured path**: set identity in `/settings`, spawn a fresh agent, repeat ‚Äî author reflects the new value, committer unchanged.
   3. **Admin commit path**: attach a fresh empty-dir repo (hits `initEmpty`) and a `master`-only repo (hits `ensureDefaultBranch`-rename / seed). `git log -1` on the seed commit shows the user's identity, not `maw@localhost`.
   4. **Banner**: unset ‚Üí banner visible; save ‚Üí banner disappears; bad email ‚Üí inline `.err`.
   5. **Reattach**: restart `maw.service`, confirm the pre-existing agent still shows its *original* author env (not retroactive).
4. Skim `commitSnapshot.ts` one more time after the change to confirm no author-side dependency slipped in.

## Out of scope

- Bulk migration of pre-existing agent_commits rows (author field is already populated from git log; no backfill needed).
- Per-role or per-project identity overrides (single global identity per user keeps the UX honest ‚Äî add later if asked).
- Auto-detecting a GitHub noreply from a GitHub OAuth link ‚Äî MAW has no GitHub OAuth today.
- Display-name/avatar/profile fields on the `users` table ‚Äî separate concern.
