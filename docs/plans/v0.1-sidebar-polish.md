# Sidebar visual + behavior polish

## Context

The new left-sidebar treeview (shipped in `v0.1-left-sidebar-treeview`) works
functionally but feels visually disconnected from the rest of the app:

- Its background (`#0b0f17`) and header strip (`#111827`) are darker than the
  page background (`#0a0a0a`), so the sidebar reads as a separate panel
  rather than part of the chrome.
- Treeview items have a heavy left connector (`border-left: 1px solid
  #1f2937` on `ul.agents` plus 1.5rem padding) that visually competes with
  the disclosure triangles and breaks, rather than reinforces, the
  Repo→Agents hierarchy.
- The collapsed sidebar is `2.5rem` wide — roomier than it needs to be,
  given it only houses the toggle button.
- A repo only appears in the tree if it currently has at least one agent.
  A user who created a repo but hasn't spawned any agents yet sees an
  empty sidebar with no path to that repo's dashboard.

This plan addresses all four.

## Approach

### 1. Match the page background, drop the panel chrome

`src/lib/client/components/RepoTreeSidebar.svelte` (CSS block, lines 188–372):

- `.sidebar` background → `#0a0a0a` (matches `:global(html, body)` at
  `src/routes/+layout.svelte:109`). Keep the right border (`#1f2937`) so
  the sidebar still has a clean edge against the content area.
- `.head` background → `transparent` (inherit page bg). Keep the bottom
  border so the header still separates from the tree.
- `.count` pill background → `#1f2937` (was `#111827`, which on the new
  bg becomes a near-black blob). Hover backgrounds (`.repo-link:hover`,
  `.agent-link:hover`, `.toggle:hover`) stay at `#1f2937`.
- `.repo-row.active` background → `#1f2937` (was `#1e293b`) so the active
  state and hover state share a palette.

### 2. Lighter, clearer hierarchy in the treeview

Same file, same CSS block:

- Drop `border-left` on `ul.agents`. Replace the visual indent with
  pure padding (`padding-left: 1.25rem`, `margin-left: 0.6rem`) so the
  agents sit under the repo name aligned with the disclosure caret.
- Use `▾`/`▸` already in place — but lighten the disclosure colour to
  `#9ca3af` so the affordance is more legible at the new bg.
- Tighten vertical rhythm: `.repo-row` padding → `0.2rem 0.25rem`,
  `.agent-link` padding → `0.2rem 0.4rem`. (Marginal but the tree feels
  cramped today around the bullets.)

### 3. Smaller collapsed width

Same file:

- `.sidebar.collapsed { width: 1.75rem; flex: 0 0 1.75rem; }` (was
  `2.5rem`). The toggle button is `1.5rem` wide; trim header padding to
  `0.25rem` when collapsed via a `.head` rule scoped under
  `.sidebar.collapsed .head { padding: 0.25rem; justify-content: center; }`
  so the chevron stays centred.

### 4. Show every repo, even with zero agents

Today `+layout.server.ts` builds the sidebar purely from
`listAgentCardsForUser` results, so empty repos are invisible. Fix at the
data layer, not the component:

- **Add** `listReposWithProjectForUser(userId)` in
  `src/lib/server/db/queries.ts` (next to `listReposForProject` at line
  130). Single query joining `repos` × `projects` filtered by
  `repos.user_id`, returning `{ id, path, project_name }` ordered by path.
  No new types — return shape mirrors `SidebarRepoNode` minus `agents`.
- **In** `src/routes/+layout.server.ts`, after building `live`/`archived`
  agent buckets, fetch the full repo list and merge: every repo gets a
  node; agents from `live` attach to it; repos with zero live agents
  still appear (with `agents: []`). Archive list is unchanged — only
  shows repos that actually have archived agents.
- **In** `RepoTreeSidebar.svelte`, the `count` pill on a repo with zero
  agents should read `0` (current code already handles this correctly —
  `repo.agents.length`). The disclosure caret should hide / be disabled
  when `repo.agents.length === 0` so users don't expand into nothing —
  render a non-button spacer of the same width to keep alignment.

No protocol or store changes; sidebar prop shape is unchanged.

## Critical files

- `src/lib/client/components/RepoTreeSidebar.svelte` — all CSS changes,
  collapsed-head padding rule, disclosure-spacer for empty repos.
- `src/routes/+layout.server.ts` — merge full repo list into
  `activeRepos`.
- `src/lib/server/db/queries.ts` — new `listReposWithProjectForUser`.
