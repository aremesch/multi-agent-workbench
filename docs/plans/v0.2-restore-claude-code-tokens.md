# Restore claude-code token usage in the archive

## Context

Token counts in the archive (per-repo at `/repos/[id]/archive` and cross-repo
at `/archive`) have shown `—` for every claude-code agent spawned in the last
~9 days. The user also reports that the agent logfile (the claude-code JSONL
transcript) is no longer surfaced. Both symptoms share one root cause.

### How tokens are *supposed* to work

1. claude-code persists every conversation as a JSONL transcript at
   `~/.claude/projects/<encoded-cwd>/<sessionId>.jsonl`. Each `assistant`
   line carries a `usage` object (`input_tokens`, `output_tokens`,
   `cache_creation_input_tokens`, `cache_read_input_tokens`).
2. To make `<sessionId>` predictable from the MAW side, the adapter spawns
   `claude --session-id <uuid>`. The same UUID is stored in
   `agents.cli_session_id` (migration `003_agent_cli_session_id.sql`).
3. The archive page resolves the deterministic file path via
   [`jsonlPathFor(cwd, sessionId)`](src/lib/server/agents/history/ClaudeJsonlTokens.ts:40)
   and sums the usage fields with
   [`summarizeTokenUsage`](src/lib/server/agents/history/ClaudeJsonlTokens.ts:48).
4. Per-row + per-repo + grand totals are aggregated in
   [`/repos/[id]/archive/+page.server.ts`](src/routes/repos/[id]/archive/+page.server.ts:117)
   and [`/archive/+page.server.ts`](src/routes/archive/+page.server.ts:76).

### What broke

Commit [`5faf4e4` — *revert(terminal): drop snapshot/dedup/JSONL/cursor-align stack*](src/lib/server/agents/AgentSupervisor.ts) (Apr 22, 2026) reverted a buggy
terminal-scrollback feature, but **the revert was overly broad** — it also
killed the unrelated `--session-id` plumbing:

- Removed `randomUUID()` mint at spawn → `cli_session_id` is hardcoded
  `null` at [AgentSupervisor.ts:559](src/lib/server/agents/AgentSupervisor.ts:559).
- Removed `cliSessionId` from `agent: { id, cliSessionId }` in
  `buildSpawnSpec` (the `BuildSpawnSpecOpts.agent` shape at
  [adapterTypes.ts:67](src/lib/shared/adapterTypes.ts:67) is now `{ id }` only).
- Removed `agent.cliSessionId` from the template-substitution map at
  [ConfigDrivenAdapter.ts:163](src/lib/server/agents/adapters/ConfigDrivenAdapter.ts:163).
- Removed `["--session-id", "{{agent.cliSessionId}}"]` from
  [cli-adapters/claude-code.jsonc](cli-adapters/claude-code.jsonc:18).

The plan note at [`docs/plans/v0.2-terminal-revert.md`](docs/plans/v0.2-terminal-revert.md)
explicitly says “`agents.cli_session_id` (migration 003) stays inert per
plan”. That was correct for the broken renderer pipeline — but the
token-summation pipeline (a different consumer of the same column) was
silently collateral damage.

### Effect

For every claude-code agent spawned after Apr 22 14:59 UTC+2:

- claude picks its own random session UUID at startup.
- MAW has no way to discover that UUID.
- `a.cli_session_id` is NULL on the row.
- The archive’s `if (a.cli_session_id)` guard never passes.
- `summarizeTokenUsage` is never called.
- Every row’s token cells render as `—`.
- Anything else that needs the JSONL path (e.g. surfacing the transcript)
  is equally blind — this is the “agent logfile not working” the user noticed.

Pre-Apr-22 archived agents still show tokens because their rows have a
populated `cli_session_id`. The regression is invisible until you scroll to
recent rows.

The MAW-owned terminal log replay (`/api/agents/[id]/log`, served from the
`terminal_log` table, used by the archive’s **View logs** button) is a
**separate** pipeline and works fine. Only the JSONL-derived data is broken.

## Recommended approach

Restore the minimal `--session-id` slice that `5faf4e4` deleted, without
re-introducing any of the renderer code that was reverted for cause.

### Trigger for minting

Auto-detect by scanning the loaded adapter config for the literal
`{{agent.cliSessionId}}` substring across `spawn.args`, `spawn.env` values,
and `spawn.initialInput`. If any reference is found, the supervisor mints a
UUID and persists it; otherwise `cli_session_id` stays NULL.

This keeps the trigger in the adapter JSONC (self-documenting), avoids
adding a redundant boolean to the schema, and degrades gracefully for
adapters that don’t need it (shell, codex, gemini today).

### File-level change list

1. **[`cli-adapters/claude-code.jsonc`](cli-adapters/claude-code.jsonc)** —
   prepend `"--session-id", "{{agent.cliSessionId}}"` to `spawn.args`. Add
   a short comment pointing at `ClaudeJsonlTokens.ts` so future maintainers
   know the link.

2. **[`src/lib/shared/adapterTypes.ts`](src/lib/shared/adapterTypes.ts:67)** —
   widen `BuildSpawnSpecOpts.agent` to `{ id: string; cliSessionId?: string | null }`.
   Optional so non-claude adapters needn’t pass it.

3. **[`src/lib/server/agents/adapters/ConfigDrivenAdapter.ts`](src/lib/server/agents/adapters/ConfigDrivenAdapter.ts)** —
   - Add `'agent.cliSessionId': opts.agent.cliSessionId ?? ''` to the
     substitution `vars` map at line 163.
   - Add a getter / readonly property `needsCliSessionId: boolean`,
     computed once in the constructor by scanning
     `cfg.spawn.args.concat(Object.values(cfg.spawn.env), [cfg.spawn.initialInput ?? ''])`
     for the literal `{{agent.cliSessionId}}` (allow optional whitespace
     inside the braces, like the substitution regex does).

4. **[`src/lib/server/agents/AgentSupervisor.ts`](src/lib/server/agents/AgentSupervisor.ts)** —
   - Re-import `randomUUID` from `node:crypto`.
   - In `spawn()`, after `const adapter = this.registry.create(role.cli_kind)`,
     compute `const cliSessionId = adapter.needsCliSessionId ? randomUUID() : null;`.
   - Pass it through `buildSpawnSpec({ ..., agent: { id: agentId, cliSessionId } })`.
   - Persist it in `insertAgent({ ..., cli_session_id: cliSessionId })` at
     [line 559](src/lib/server/agents/AgentSupervisor.ts:559).
   - Browser-agent fast path at [line 480](src/lib/server/agents/AgentSupervisor.ts:480)
     stays `cli_session_id: null` (no CLI to talk to).

5. **Tests** — extend
   [`ConfigDrivenAdapter.test.ts`](src/lib/server/agents/adapters/ConfigDrivenAdapter.test.ts)
   with two cases:
   - Adapter that references `{{agent.cliSessionId}}` reports
     `needsCliSessionId === true` and substitutes the supplied value.
   - Adapter that doesn’t reference it reports `false` and leaves args
     unchanged when no `cliSessionId` is passed.
   Then add an `AgentSupervisor` test (or extend an existing spawn test)
   asserting that spawning a claude-code role writes a non-null UUID into
   `agents.cli_session_id` and that the value matches the one passed via
   `--session-id` in the recorded spawn args.

### Out of scope (intentionally)

- **No back-fill of legacy NULL rows.** Agents archived between Apr 22 and
  the fix used unknown-to-MAW session UUIDs; the JSONL files exist on disk
  but cannot be reliably matched to specific agent rows (multiple agents
  may have run in the same worktree). They will keep showing `—`. Mention
  this in the eventual commit message; do not attempt heuristic matching.
- **No re-introduction of the JSONL→ANSI renderer**, `historySource`,
  `scrollbackMode`, `forceRedrawOnReconnect`, or any other piece of the
  reverted stack. The pane-snapshot reconnect from
  [`v0.2-terminal-pane-snapshot.md`](docs/plans/v0.2-terminal-pane-snapshot.md)
  is the current and correct terminal-restore path.
- **No new UI** to surface the JSONL transcript path itself. If the user
  wants a “open transcript” affordance on the archive page, that is a
  follow-up plan — this plan only restores token aggregation.
- Codex and Gemini adapters get no change; they don’t have JSONL readers
  yet. When they do, those adapter JSONCs flip on `{{agent.cliSessionId}}`
  themselves and the supervisor logic handles them automatically.

## Verification

1. `pnpm test` — new `ConfigDrivenAdapter` cases + the `AgentSupervisor`
   spawn case must pass; existing 472-case suite must stay green.
2. `pnpm check` — types must stay 0/0.
3. `pnpm build` — bundled `build/server.js` must build (proves the
   externalisation chain still works after the supervisor edit).
4. Live smoke against the dev server:
   - Spawn a claude-code agent against a real repo (any role).
   - Verify the row’s `cli_session_id` is non-null:
     `sqlite3 ~/.local/share/maw/maw.db "select id, cli_kind, cli_session_id from agents order by created_at desc limit 5;"`
   - Verify claude wrote the JSONL at the expected path:
     `ls -la ~/.claude/projects/$(printf %s "$(realpath <worktree>)" | sed 's,[/.],-,g')/<uuid>.jsonl`
   - Have the agent run at least one tool call so `usage` appears, then
     exit it (`tmux send-keys exit` or close the modal).
   - Reload `/repos/<id>/archive` and confirm the new row’s **Tokens**
     column shows non-zero values; reload `/archive` and confirm the
     per-repo + grand totals also include the new contribution.
5. Regression sanity: spawn a `shell` smoke agent (which does NOT
   reference `{{agent.cliSessionId}}`) and confirm its row’s
   `cli_session_id` stays NULL and tokens render `—` as before.
