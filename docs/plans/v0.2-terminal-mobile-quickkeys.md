# Remove terminal input field; add configurable mobile quick-keys

## Context

Under every `<xterm>` in the agent dialog we render a free-text
`<input>` + **Send** button (`AgentTerminalPanel.svelte` lines 186ÔÇô193).
It's redundant on the **webapp** ÔÇö the terminal itself accepts all
keystrokes via `term.onData` Ôćĺ `send_keys` Ôćĺ `tmux send-keys -l`, and
typing in the input then pressing **Send** just fires the same bytes
one-shot via `sendInput`. So on desktop it's dead weight.

On the **smartphone PWA** the dialog has the opposite problem: the soft
keyboard on iOS/Android doesn't expose **cursor up/down**, **Shift+Tab**,
**Esc** or **Ctrl+C** ÔÇö keys the CLIs (especially Claude Code) need
constantly (history recall, menu navigation, exiting a thinking block,
aborting). The user can't navigate a Claude menu without arrow keys.

This plan does two things:

1. **Webapp**: delete the input field + Send button outright. xterm is
   the only input surface.
2. **Mobile**: render a compact row of **quick-key buttons** under
   xterm, driven by a new per-adapter `mobileQuickKeys` array in the
   CLI-adapter JSONC. Each button fires its configured bytes through
   the existing `send_keys` path and refocuses xterm.

The "show quick-keys" trigger must approximate "touch device with no
physical keyboard attached", with a per-user override for edge cases.

## Detection (touch + per-user override)

Two signals combine:

- **Touch signal** (reactive CSS media query):
  `matchMedia('(pointer: coarse)')` ÔÇö primary pointer is a finger,
  i.e. a phone or a tablet.
- **User override** (per-user setting, persisted in the same store as
  `ui.sidebar.collapsed`): `ui.mobileQuickKeys` Ôłł `{auto, always, never}`,
  default `auto`.

Effective rendering condition in `AgentTerminalPanel`:

```ts
const show =
  quickKeys.length > 0 &&
  (mode === 'always' ||
    (mode === 'auto' && mqTouch.matches));
// mode === 'never' always hides.
```

So the behaviour the user asked for:

- Touch device, setting left at default (`auto`) Ôćĺ bar shows.
- Touch device, setting set to `never` Ôćĺ bar hidden.
- Non-touch device (desktop browser), setting `always` Ôćĺ bar shows
  (useful for dev verification and edge-case users).
- Non-touch device, setting `auto` or `never` Ôćĺ bar hidden.

The setting row lives under
[/settings](src/routes/settings/+page.svelte) with three radio options
(Auto / Always / Never). No attempt to detect "physical keyboard
attached" ÔÇö that isn't possible on the web platform. `auto` mode
trusts the touch signal alone; anyone who wants it off flips the
setting once per account.

## Adapter schema change (`mobileQuickKeys`)

New optional field on the adapter JSONC. Empty array by default
(adapter opts out of quick-keys).

```jsonc
"mobileQuickKeys": [
  { "id": "up",         "label": "ÔćĹ",     "keys": "\u001b[A" },
  { "id": "down",       "label": "Ôćô",     "keys": "\u001b[B" },
  { "id": "shift-tab",  "label": "ÔçžTab",  "keys": "\u001b[Z" },
  { "id": "esc",        "label": "Esc",   "keys": "\u001b"   }
]
```

Field shape (Zod + TS):

| field   | type   | notes                                                                  |
|---------|--------|------------------------------------------------------------------------|
| `id`    | string | stable key (for logs / overrides). `[a-z0-9-]+`, unique per adapter.   |
| `label` | string | short on-button text. Unicode glyphs allowed (`ÔćĹ`, `ÔçžÔçą`).             |
| `keys`  | string | raw UTF-8 bytes to forward via `send_keys`. Literal escapes OK (`\x1b[A`). |

`keys` is injected as-is into the existing `send_keys` WS frame
(`AgentTerminalPanel.onTerminalData`) ÔÇö no new server path, no new
encoding.

## Per-adapter presets (populate all four)

All labels use single unicode glyphs to keep buttons finger-tip-sized.

**[claude-code.jsonc](cli-adapters/claude-code.jsonc)**
```
ÔćĹ   \x1b[A       cursor up      (history / menu up)
Ôćô   \x1b[B       cursor down    (history / menu down)
ÔçžÔçą  \x1b[Z       shift+tab      (reverse cycle plan/thinking tabs)
Esc \x1b         escape         (dismiss / stop streaming)
```

**[shell.jsonc](cli-adapters/shell.jsonc)**
```
ÔćĹ   \x1b[A       readline history prev
Ôćô   \x1b[B       readline history next
Ôçą   \t            tab completion
Esc \x1b
^C  \x03          SIGINT
```

**[codex.jsonc](cli-adapters/codex.jsonc)**
```
ÔćĹ   \x1b[A
Ôćô   \x1b[B
Ôçą   \t
Esc \x1b
^C  \x03
```

**[gemini.jsonc](cli-adapters/gemini.jsonc)**
```
ÔćĹ   \x1b[A
Ôćô   \x1b[B
Ôçą   \t
Esc \x1b
^C  \x03
```

These are sensible defaults; each JSONC can be tuned independently
because the registry hot-reloads.

  path as xterm's own keystrokes.

  Orthogonal; tackle when the push-notification PWA scope lands.
