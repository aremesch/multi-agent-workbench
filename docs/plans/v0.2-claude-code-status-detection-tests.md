# v0.2 — claude-code CLI status detection tests

## Context

The claude-code adapter (`cli-adapters/claude-code.jsonc`) ships six regex
patterns plus a `cursor_at_prompt` idle detector that together drive the
agent state machine: `BOOTING → READY → WORKING → WAITING_PROMPT → IDLE`,
with `EXITED` reached via the tmux `session-closed` hook. Today the regex
engine has thorough generic coverage in
[`ConfigDrivenAdapter.test.ts`](src/lib/server/agents/adapters/ConfigDrivenAdapter.test.ts),
but **none of those tests load the real claude-code config**, so a
regression that breaks one of the claude-code patterns won't fail the
suite. `AgentRuntime` — the layer that translates adapter events into
`agents.status` DB updates and routes keystrokes into tmux — has zero
unit tests at all.

This change adds three layers of coverage for the lifecycle the user
described (boot → first prompt → "write 10 random lines" → multi-choice
prompt answered with cursor keys → exit detected):

1. **Synthetic adapter test** — fast, deterministic, always-on. Verifies the
   real claude-code regex shapes against crafted byte streams.
2. **AgentRuntime test** — fast, mocked tmux/FIFO/DB. Verifies status-flip
   translation and the tmux input plumbing (`enqueueInput`,
   `enqueueRawKeys`, `enqueueAnswer`).
3. **Live integration test** — spawns the real `claude` binary via tmux
   using **Haiku** for cost (`--model claude-haiku-4-5-20251001`).
   Drives the full lifecycle. Auth comes from the local `claude`
   install (OAuth/keychain — same mechanism normal MAW uses); MAW
   never owns an Anthropic key. Gated on `claude` being on `$PATH`;
   not part of the default `pnpm test` run.

## Approach

### File 1: `claude-code-lifecycle.test.ts` (synthetic, always-on)

Pattern is the existing `ConfigDrivenAdapter.test.ts` plus a real-config
loader that reuses the parser logic from
[`scripts/test-adapter.ts`](scripts/test-adapter.ts):

```ts
function loadClaudeCodeAdapter(): ConfigDrivenAdapter {
  const path = join(process.cwd(), 'cli-adapters/claude-code.jsonc');
  const cfg = adapterConfigSchema.parse(parseJsonc(readFileSync(path, 'utf8')));
  return new ConfigDrivenAdapter(cfg);
}
```

Cases:

- **boot → working → ready**: fresh adapter is BOOTING; banner bytes flip to WORKING (heuristic); `\n> \n` fires `ready` event, state READY. With `vi.useFakeTimers`, advance 2.5 s past last ingest → `isIdleWaiting() === true` (cursor_at_prompt + 2 000 ms).
- **plan_approval_prompt**: feed `"…\n  1. Yes\n  2. No\nWould you like to proceed?"` → one `prompt_detected` event with `patternId === 'plan_approval_prompt'`, `choices` deep-equals `['1','2','abort']`, state WAITING_PROMPT, `isIdleWaiting() === true`.
- **tool_permission_prompt** (named groups): feed `"Do you want to run Bash(ls /tmp)?"` → `detail.tool === 'Bash'`, `detail.args === 'ls /tmp'`.
- **shell_command_prompt**: feed `"Run npm test ? [y/N]"` → `detail.cmd === 'npm test'`.
- **task_done**: feed `"✓ Task complete"` → state IDLE; second feed within tail returns `[]` (firedMatches dedup).
- **rate_limit error**: feed `"Encountered rate-limit"` → `error` event, state unchanged.
- **dedup eviction**: feed prompt → 9 KB filler → re-feed prompt → re-fires (eviction works).
- **input.answerPrompt** for every documented choice: `'1'`/`'2'` → `[<digit>,'Enter']`; `'yes'` → `['y','Enter']`; `'no'` → `['n','Enter']`; `'abort'` → `['C-c']` (no Enter — important); fallback `'foo'` → `['foo','Enter']`.
- **mobileQuickKeys VT220 bytes**: assert `mobileQuickKeys` from the loaded config carries exactly `'[A'` / `'[B'` / `'[Z'` / `''` for `up`/`down`/`shift-tab`/`esc`. Locks the escape codes against accidental edits.
- **needsCliSessionId === true** (the `--session-id {{agent.cliSessionId}}` template).

### File 2: `AgentRuntime.test.ts` (mocked, always-on)

First-ever test for `AgentRuntime`. Bootstraps:

- `vi.mock('../tmux/TmuxSession.js')` → stub `Tmux.sendLiteral`, `Tmux.sendKey`, `Tmux.resizeWindow` as `vi.fn().mockResolvedValue(undefined)`.
- `vi.mock('../tmux/FifoStreamer.js')` → constructor returns a tiny EventEmitter shape (`onChunk` registration, no real FD).
- `vi.mock('../bootstrap.js')` → `getPushService()` returns `{ sendToUser: vi.fn() }`.
- DB: `openMemoryDb()` from [`tests/unit/helpers/db.ts`](tests/unit/helpers/db.ts); seed user/repo/role/agent rows.

Cases:

- **status flips** for each `processEvent` kind:
  - `ready` → `agents.status = 'running'` and `'state'` event with `'running'`.
  - `prompt_detected` → `'waiting_input'`, `attention_needed_at` set.
  - `task_done` → `'idle'`.
  - `exited` (via `finishAsExited`) → `'exited'`.
- **enqueueInput("write 10 random lines", true)** → `Tmux.sendLiteral(session, 'write 10 random lines')` then `Tmux.sendKey(session, 'Enter')`, in order.
- **enqueueInput(text, false)** → no `sendKey` call.
- **enqueueRawKeys('[A[A[B')** → `Tmux.sendLiteral` called once with the literal three-escape-sequence string; `sendKey` not called.
- **enqueueRawKeys('')** → no tmux calls.
- **enqueueAnswer('1')** with the real claude-code adapter wired in → `sendLiteral('1')` + `sendKey('Enter')`.
- **enqueueAnswer('abort')** → `sendKey('C-c')` only.
- **input queue serialisation**: dispatch three `enqueueInput` calls without awaiting; `Tmux.sendLiteral.mock.calls` order matches dispatch order.
- **alert dedup window**: two regex `prompt_detected` events with same `patternId+raw` while `lastHookAt` was set within `HOOK_PRIORITY_WINDOW_MS=30 s` → only the first writes an alert (covers the 30 s suppression window from
  [`AgentRuntime.ts:69`](src/lib/server/agents/AgentRuntime.ts:69)).

### File 3: `claude-code-live.test.ts` (real subprocess, gated)

Skip-gate at file top — checks for the `claude` binary, **not** for an
API key. MAW never holds an Anthropic key; `claude` self-authenticates
via the same mechanism a normal MAW install uses (OAuth/keychain on the
host that ran `claude login`):

```ts
import { execaSync } from 'execa';
function claudeAvailable(): boolean {
  try { execaSync('which', ['claude']); return true; } catch { return false; }
}
const liveTest = claudeAvailable() ? test : test.skip;
```

Setup per test:

1. `mkdtemp` a scratch dir, `git init -b main` + empty commit.
2. `openMemoryDb()` → seed user/repo/role + agent row with a fresh ULID.
3. Build a `ConfigDrivenAdapter` from `cli-adapters/claude-code.jsonc`.
4. Construct `AgentRuntime` directly (bypassing the supervisor) with the agent row + adapter + a `mkdtemp` fifo dir.
5. Spawn via `Tmux.newSession({ session: 'maw-agent-<id>', command: 'claude', args: ['--model', 'claude-haiku-4-5-20251001', '--session-id', cliSessionId, '--append-system-prompt', 'Always answer in one short sentence.'], env: {}, cwd: scratchDir })` — no API-key env injection; `claude` reads its own credentials.
6. Wire `FifoStreamer` → `runtime.ingest(chunk)` (via the existing public surface).
7. Register `runtime.on('state', ...)` and `runtime.on('event', ...)` listeners that push to arrays the test polls.

Helpers:

- `await waitForState('running', 30_000)` — polls the listener array.
- `await waitForEvent('prompt_detected', 30_000)` — polls events.

Test scenario (single sequential `liveTest`, 90 s timeout):

```
1. Spawn → expect state === 'running' (READY mapped).
2. Sleep 2.5 s → adapter.isIdleWaiting() === true.
3. enqueueInput("write 10 random lines to a file called notes.txt", true)
   → expect state event for 'running' or 'waiting_input' shortly after.
4. Wait for prompt_detected event:
   - patternId === 'tool_permission_prompt' (claude asks to write).
   - detail.tool matches /Write|Edit|Bash/.
   - state === 'waiting_input', DB status flipped.
5. Cursor-key path: enqueueRawKeys('[B') (move down once).
   - Inspect tmux capturePane after 200 ms; assert pane content shows
     the choice indicator moved (best-effort visual check; falls back
     to asserting the raw bytes were forwarded via Tmux.sendLiteral).
6. enqueueAnswer('2') → claude declines, returns to ready.
   - Wait for state === 'idle' or 'running' (ready re-fires).
7. Exit: send `Tmux.sendKey('C-c')` ×2 (claude's standard quit) OR
   enqueueInput('/exit', true) — whichever the adapter handles cleaner.
8. Expect state === 'exited' via the runtime's tmux session-closed wait.
9. Expect DB agents.status === 'exited'.
```

Teardown: `Tmux.killSession`, `rm -rf` scratch + fifo dirs, close DB.

Cost & cadence:
- Haiku 4.5 input ~$1/MTok, output ~$5/MTok. One run ~500/300 tokens → < $0.005, billed against whichever account the local `claude` install is logged into.
- Default `pnpm test` excludes the `integration` project so unit-test runs stay free and offline.
- Run live locally with `pnpm test:integration` (anywhere `claude` is installed and logged in). CI gates on a separate workflow (out of scope for this plan — follow-up if desired).

### Why this layering

- File 1 fails fast when `claude-code.jsonc` regexes regress, without
  needing a network or a key.
- File 2 fails fast on `processEvent` / input-queue regressions in
  `AgentRuntime`, without tmux.
- File 3 catches regressions in the **integration** of the real claude
  TUI with our adapter regex (e.g. claude renames its plan-approval
  text and our regex stops matching — only the live test catches this).
