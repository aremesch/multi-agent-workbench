# v0.2 — Configurable Spawn Args (skip-permissions toggle)

## Context

Claude Code's `--dangerously-skip-permissions` flag is currently
hardcoded in `cli-adapters/claude-code.jsonc`. The owner wants it on by
default (since they trust their agents) but other users may want
permission prompts enabled — especially once Phase B push notifications
let them approve from their phone.

This plan adds **per-user default spawn settings** (stored in
`user_settings`) and a **per-spawn override** in the spawn form, so
every agent spawn can choose its own flags. The adapter JSONC files stay
read-only; user preferences are merged at spawn time.

## Design

### 1. Adapter schema: declare optional args

Add an `optionalArgs` array to the adapter spawn config. Each entry
describes a CLI flag the user can toggle on or off.

**File: `cli-adapters/claude-code.jsonc`** — spawn block becomes:

```jsonc
"spawn": {
  "command": "claude",
  "args": ["--session-id", "{{agent.cliSessionId}}"],
  "optionalArgs": [
    {
      "id": "skip-permissions",
      "flag": "--dangerously-skip-permissions",
      "label": "Skip permission prompts",
      "description": "Agent runs tools without asking. Disable to get push notifications for each tool use.",
      "default": false
    }
  ],
  "env": { "ANTHROPIC_API_KEY": "{{env.ANTHROPIC_API_KEY}}" },
  "initialInput": "{{task.body}}"
}
```

Key points:
- `--dangerously-skip-permissions` is **removed** from the fixed `args`
  array and moved to `optionalArgs`.
- `default: false` is the safe default for new users.
- Each entry has an `id` (stable key for settings), `flag` (the actual
  CLI arg), `label`/`description` (UI text), and `default` (boolean).

### 2. Schema changes

**File: `src/lib/server/agents/adapters/adapter.config.schema.ts`**

Add to the `spawn` object:

```typescript
optionalArgs: z.array(z.object({
  id: z.string().min(1),
  flag: z.string().min(1),
  label: z.string().min(1),
  description: z.string().optional(),
  default: z.boolean().default(false)
})).default([])
```

Also update `schemas/adapter.schema.json` to match (DX autocomplete).

### 3. User settings: per-adapter defaults

**Setting key:** `spawn.defaults.<cli_kind>` (e.g.,
`spawn.defaults.claude-code`)

**Value:**

```json
{ "optionalArgs": { "skip-permissions": true } }
```

This is stored in the existing `user_settings` table via
`setUserSetting()` / `getUserSetting()`. No migration needed.

**New API endpoint: `PUT /api/user/spawn-defaults`**

**File: `src/routes/api/user/spawn-defaults/+server.ts`** (new)

```
PUT { cliKind: string, optionalArgs: Record<string, boolean> }
→ setUserSetting(userId, `spawn.defaults.${cliKind}`, JSON.stringify({ optionalArgs }))
→ 204
```

**New query helper:**

**File: `src/lib/server/db/queries.ts`** — add:

```typescript
export function getSpawnDefaults(userId: string, cliKind: string):
  { optionalArgs: Record<string, boolean> } | null
```

Reads `spawn.defaults.<cliKind>` from `user_settings`, parses JSON.

### 4. Settings page: spawn defaults section

**File: `src/routes/settings/+page.svelte`**

Add a new "Agent Defaults" section below the existing theme/locale
sections. For each registered CLI kind that has `optionalArgs`, show a
group of toggles:

```
┌─ Agent Defaults ──────────────────────────────┐
│                                                │
│  Claude Code                                   │
│  ┌──────────────────────────────────────────┐  │
│  │ ☑ Skip permission prompts                │  │
│  │   Agent runs tools without asking.       │  │
│  │   Disable to get push notifications      │  │
│  │   for each tool use.                     │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  Codex CLI                                     │
│  (no optional flags)                           │
│                                                │
└────────────────────────────────────────────────┘
```

Each toggle fires `PUT /api/user/spawn-defaults` on change. The
server-side `+page.server.ts` loads the current defaults to hydrate the
initial state.

**File: `src/routes/settings/+page.server.ts`** (new or extend existing)

Load:
- Adapter registry list (kind + displayName + optionalArgs metadata)
- User's current `spawn.defaults.*` settings for each kind

Pass to the page as `data.spawnDefaults`.

### 5. Spawn form: per-agent override

**File: `src/lib/client/components/SpawnAgentForm.svelte`**

When the user selects a role, resolve the `cli_kind` → look up the
adapter's `optionalArgs`. If any exist, render a collapsible "Advanced"
section below the task body textarea with toggles pre-filled from the
user's defaults (or the adapter's `default` if no user setting exists).

```
┌─ Spawn Agent ─────────────────────────────────┐
│  Role:  [Claude Code ▾]                        │
│  Repo:  [my-project ▾]                         │
│  Task:  [___________________________]          │
│  Body:  [___________________________]          │
│                                                │
│  ▸ Advanced                                    │
│    ☑ Skip permission prompts                   │
│                                                │
│            [Cancel]  [Spawn]                   │
└────────────────────────────────────────────────┘
```

The toggle values are submitted as hidden form fields:
`optionalArgs[skip-permissions]=true`.

### 6. Spawn action: merge optional args

**File: `src/routes/agents/new/+page.server.ts`**

In the form action, after extracting role/repo/task fields:

1. Parse `optionalArgs[*]` from form data.
2. Pass them through to `supervisor.spawn()` as a new
   `optionalArgs: Record<string, boolean>` field on `SpawnAgentArgs`.

**File: `src/lib/server/agents/AgentSupervisor.ts`**

In `spawn()`, pass `optionalArgs` to `adapter.buildSpawnSpec()`.

**File: `src/lib/server/agents/adapters/ConfigDrivenAdapter.ts`**

In `buildSpawnSpec()`, after building the base `args` array from
`this.cfg.spawn.args`, append flags from `optionalArgs`:

```typescript
// Merge optional args
for (const opt of this.cfg.spawn.optionalArgs) {
  const enabled = opts.optionalArgs?.[opt.id] ?? opt.default;
  if (enabled) {
    spec.args.push(opt.flag);
  }
}
```

### 7. Data flow for spawn form

The spawn form needs to know the `optionalArgs` metadata for the
selected CLI kind plus the user's defaults. Two options:

**Option A (recommended):** Load adapter metadata in the page's
`+page.server.ts` (or layout server load) and pass it as page data.
The spawn form already receives `cliKinds: CliKindOption[]` — extend
`CliKindOption` to include `optionalArgs` metadata. Load user defaults
alongside.

**File: `src/routes/(app)/+layout.server.ts`** (or wherever
`cliKinds` is loaded)

```typescript
// Extend CliKindOption
interface CliKindOption {
  kind: string;
  displayName: string;
  optionalArgs: Array<{
    id: string;
    flag: string;
    label: string;
    description?: string;
    default: boolean;
  }>;
}
```

Also load user spawn defaults and pass as `data.spawnDefaults`:

```typescript
Record<string, { optionalArgs: Record<string, boolean> }>
```

The SpawnAgentForm receives both and can derive the initial toggle
state for the selected role's `cli_kind`.

## Verification

1. **`pnpm check`** — zero errors.
2. **Settings page:** toggle "Skip permission prompts" on for Claude
   Code → reload → toggle persists. Toggle off → reload → persists.
3. **Spawn form:** select a Claude Code role → expand Advanced → toggle
   is pre-filled from user setting. Override it → spawn → verify the
   tmux session was started with or without
   `--dangerously-skip-permissions` by checking `tmux list-panes -t
   maw-agent-<id> -F '#{pane_start_command}'` or inspecting the agent's
   terminal output.
4. **Non-Claude adapters:** Codex/Gemini/shell have no `optionalArgs` →
   Advanced section is hidden or empty. Spawn works as before.
5. **Default behavior:** a fresh user with no settings sees the adapter's
   `default` value (false for skip-permissions) → Claude Code spawns
   **with** permission prompts.
