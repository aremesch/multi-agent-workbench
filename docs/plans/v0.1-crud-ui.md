## Context

Two things are needed before and during the next implementation step:

1. **Housekeeping** — the workspace is not yet under version control, has no `CLAUDE.md` to carry forward guidelines and status between sessions, and no `docs/plans/` to persist the roadmap. These need to exist before we continue, so every subsequent change lands on top of a clean committed baseline.

User-confirmed decisions:
- **Full CRUD UI** (not a minimal spawn-only form).
- **New branch `maw/<agentId>` from HEAD** — safe isolation per agent.
- **`CLAUDE.md` stays lean**: intro, short current-status, Guidelines, a pointer to `docs/plans/`. No completed-task logs — git history covers that.
- **Only MAW-related plans** get copied to `docs/plans/`. The unrelated `abundant-wobbling-harbor.md` (portfolio tracker) in `~/.claude/plans/` is not relevant to this repo and must not be copied.

## Phase 2: CRUD UI (after housekeeping is committed)

### Pattern: SvelteKit form actions throughout

No `+server.ts` REST endpoints. Form actions give us free CSRF, free progressive enhancement, and match the existing `login/+page.server.ts` pattern.

### Registry access from routes

`hooks.server.ts` puts `supervisor` on `event.locals`. The supervisor already holds `.registry`, so routes needing the list of adapter kinds do `locals.supervisor.registry.list()`. No hook changes needed.

### Fix: agent-id lock-step with worktree

`AgentSupervisor.spawn()` at `src/lib/server/agents/AgentSupervisor.ts:95` currently generates its own `agentId` inside the function. The caller needs that id up front to name the worktree dir and branch. Refactor: add `agentId: string` to `SpawnAgentArgs`, drop the inline `ulid()` call inside `spawn()`. Caller pre-generates the id, uses it for worktree creation, then hands it to `spawn()`.

### Smoke-test adapter (no third-party CLI needed)

Add `cli-adapters/shell.jsonc` — a minimal adapter that launches `bash -i`. Zero pattern matches, no prompts, no task interpolation. Lets us verify the whole tmux / FIFO / WS pipeline end-to-end on any machine without depending on `claude` / `codex` / `gemini` being installed.

### Routes to create

All use form actions; no `+server.ts`.

| Route | Purpose |
|---|---|
| `src/routes/projects/new/+page.{svelte,server.ts}` | name, default_branch → `insertProject` → redirect to `/projects/[id]` |
| `src/routes/projects/[id]/+page.{svelte,server.ts}` | Project detail: list repos, link to add repo |
| `src/routes/projects/[id]/repos/new/+page.{svelte,server.ts}` | path, origin_url → validate via `git rev-parse --git-dir` → `insertRepo` |
| `src/routes/roles/+page.{svelte,server.ts}` | List roles + "new role" link |
| `src/routes/roles/new/+page.{svelte,server.ts}` | name, cli_kind select (from `registry.list()`), system_prompt → `insertRole` |
| `src/routes/agents/new/+page.{svelte,server.ts}` | role, repo, optional task → worktree + spawn → redirect to `/agents/[id]` |

### Files to modify

- `src/routes/+page.server.ts` — also load `listRoles(userId)` for the dashboard cards.
- `src/routes/+page.svelte` — three-card layout (Projects / Roles / Agents), each with a "new" link, plus a single "Spawn agent" CTA.
- `src/lib/server/agents/AgentSupervisor.ts` — accept `agentId` in `SpawnAgentArgs`; stop generating it inline.
- `cli-adapters/shell.jsonc` — new bash smoke adapter.

### Server-side validation

Every action returns `fail(400, { fields, error })` on invalid input (matches login pattern).

- **Project**: `name` non-empty, `default_branch` matches `^[\w./-]+$`
- **Repo**: `path` absolute, exists on disk, `git -C <path> rev-parse --git-dir` succeeds
- **Role**: `name` non-empty, `cli_kind` ∈ `registry.list()`
- **Agent spawn**: role_id + repo_id belong to the current user; worktree creation must succeed before inserting the agent row

### Existing code reused (no duplication)

- Queries in `src/lib/server/db/queries.ts`: `insertProject`, `insertRepo`, `insertWorktree`, `insertRole`, `listProjects`, `listReposForProject`, `listRoles`, `getProject`, `getRepo`, `getRole`
- `WorktreeManager.create()` — `src/lib/server/git/WorktreeManager.ts:42`
- `AgentSupervisor.spawn()` — `src/lib/server/agents/AgentSupervisor.ts:95`
- `AdapterRegistry.list()` — `src/lib/server/agents/adapters/AdapterRegistry.ts:80`
- `resolveSession` via `locals.user` — `hooks.server.ts`
- Form-action pattern — `src/routes/login/+page.server.ts`
