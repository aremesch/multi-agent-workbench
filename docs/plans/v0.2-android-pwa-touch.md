# v0.2 — Android PWA touch & thumbnail fixes

## Context

Two regressions surface only when the PWA is opened on an Android phone:

1. **Touch scroll dead inside the agent terminal modal** (and inside any
   xterm.js view, including the archive log replay). On desktop the
   xterm scrollback scrolls fine with the wheel, but a finger swipe on
   Android never moves the viewport.
2. **Agent thumbnails on the dashboard render blank.** Cards on
   `/dashboard` and `/repos/[id]` show only the gridstack resize handle
   in the lower-right corner — no border, no header, no snapshot.

Both bugs make the phone-first half of MAW's product goal unusable.
The fixes are small and self-contained: tell xterm's viewport to honor
vertical pan gestures, and stop relying on a single CSS feature
(`container-type: size` + `cqw`/`cqh`) to size the thumbnail font.

## Root causes

**Bug 1 — xterm touch scroll.** xterm.js renders a transparent
`.xterm-viewport` over its canvas as the actual scroll container. On
touch devices Chrome's default `touch-action: auto` lets the swipe
become a scroll, but the surrounding host
(`src/lib/client/components/Terminal.svelte:171` —
`.terminal-host { overflow: hidden }`) and the modal wrapper consume
the gesture before it reaches the viewport. xterm also leaves
`touch-action` unset on `.xterm-viewport`, so Android sometimes
classifies the swipe as a horizontal pan / zoom hint and discards it.

**Bug 2 — thumbnail blank on Android.**
`AgentCard.svelte:298-301` sizes the snapshot font with
`min(calc(100cqw / (cols*0.6)), calc(100cqh / rows))` on a
`container-type: size` body. When gridstack lays out on a narrow
Android viewport the body's container size can resolve to 0 during the
first paint (cells are still being measured), so the font collapses to
0 and nothing is visible. Even after layout settles, Android Chrome's
container-query implementation does not re-evaluate the font when the
container box updates inside a `position: absolute` gridstack child as
reliably as desktop Chrome does. The result: blank `<pre>`, blank
placeholder area, and because `.agent-card` is `overflow: hidden` with
a 1px border, the whole card visually disappears behind the gridstack
item — only the gridstack-injected resize handle (a separate sibling)
remains visible.

## Changes

### 1. `src/lib/client/components/Terminal.svelte`
- Add `touch-action: pan-y` on `.terminal-host` and on the
  `:global(.xterm-viewport)` selector so Android forwards vertical
  swipes to xterm's scroller.
- Drop `overflow: hidden` from `.terminal-host` (xterm's own viewport
  already clips); keep `min-width/min-height: 0` for the flex parent.
- In the `Terminal` constructor options pass `scrollOnUserInput: true`
  (xterm v5+ default is true, but make it explicit so it cannot be
  flipped by future option-merge changes).

### 2. `src/lib/client/components/AgentCard.svelte`
- Replace the container-query font-size with a JS-measured size:
  - Add a `ResizeObserver` on the `.body` element that reads
    `clientWidth` / `clientHeight` and writes
    `--thumb-font: <px>px` as an inline style on `.snapshot`.
  - Compute `font = Math.max(2, Math.min(width / (cols*0.6), height / rows))`.
  - Remove `container-type: size` and the `100cqw / 100cqh` rule.
- Keep the existing `--cols` / `--rows` CSS vars for read-only debug;
  the actual sizing happens in JS.
- Tear down the ResizeObserver in `onDestroy` alongside the existing
  poll timer.

### 3. (optional, same PR) gridstack touch drag/resize
Currently `AgentGrid.svelte:128` initialises gridstack with no touch
options. v12 supports touch natively but needs
`alwaysShowResizeHandle: 'mobile'` so the handle is reachable, and the
default `column: 12` is unusable on a 400 px phone. Add a small
responsive tweak:
- `columnOpts: { breakpoints: [{ w: 600, c: 1 }] }` so phones get a
  single-column stack.
- `alwaysShowResizeHandle: 'mobile'`.

Flag this as a separate commit so it can be reverted independently if
mobile drag turns out to need more work.

## Verification

1. `pnpm check` clean (0/0).
2. `pnpm build && node build/server.js`, open the PWA on an Android
   phone (Chrome and the installed PWA both):
   - Dashboard shows agent thumbnails with header, border, and live
     snapshot text. Resize a card; thumbnail re-fits with no overflow.
   - Open an agent terminal; finger-swipe up/down inside the terminal
     scrolls the xterm scrollback. Tap selection still works.
   - Archive → View logs replay also scrolls by touch.
3. Desktop regression sweep: the same dashboard and terminal modal
   behave identically to before on Chrome/Firefox/Safari (wheel scroll,
   font-size still fills the card, no layout flicker on resize).
4. Lighthouse PWA audit on the dashboard page does not regress.
