# v0.2 — Dashboard grip handle (fix mobile scroll-vs-drag)

## Context

On the PWA / phone, scrolling through the agent dashboard is broken:
as soon as a finger lands on a tile, gridstack's drag-to-rearrange
captures the gesture and the tile follows the finger around instead of
the page scrolling.

Root cause: gridstack v12 is currently configured with
`draggable: { handle: '.agent-card-header' }`
([`AgentGrid.svelte:146`](src/lib/client/components/AgentGrid.svelte:146)).
The `.agent-card-header` strip is a full-width bar that occupies the
top of every tile
([`AgentCard.svelte:194-203`](src/lib/client/components/AgentCard.svelte:194)).
At `c: 1` (single-column on phones, < 600 px), every tile stacks
vertically, so headers form a series of horizontal bars directly in
the thumb's path during a vertical swipe — which is exactly how mobile
users start a scroll. Gridstack puts `touch-action: none` on the
configured handle and `preventDefault`s touchmove, so the browser
never gets to scroll.

Goal: make the dashboard scroll naturally on touch while keeping
drag-to-rearrange working for everyone (desktop + power users on
tablets/phones).

## Approach

Move the drag handle off the full-width header strip and onto a small
**grip icon** rendered inside the header. Gridstack drags only from
the grip; headers and bodies scroll natively on touch and behave like
plain text/buttons on desktop. This is the standard mobile UI for
reorderable lists (Trello, kanban, iOS settings reorder).

This is a single-affordance change for *all* devices — desktop users
will also need to grab the grip to drag, instead of grabbing the
header anywhere. The grip is small but visually distinct (low-contrast
six-dot pattern, `⠿`), so the affordance reads correctly on both
platforms.

Why not auto-disable drag on touch (the user-toggle approach mirroring
`ui.mobileQuickKeys`): preserves zero functionality on touch and adds
~150 LoC + i18n in four locales for a fix that the grip-icon
approach solves with a single visual change. Rejected.

## Changes

### 1. [`src/lib/client/components/AgentCard.svelte`](src/lib/client/components/AgentCard.svelte)

Add a `.agent-card-grip` element inside `.agent-card-header` (right
side, after the status badge, or left, before the titles — pick
whichever the visual review prefers; left edge is more conventional
for "reorder" affordance). The grip:

- Is a `<span>` (decorative, not focusable — drag-and-drop is a mouse
  affordance and we don't expose keyboard reorder anyway).
- Has `aria-hidden="true"` and renders `⠿` (U+28FF Braille pattern,
  same character used in many drag-handle affordances) or a small SVG.
- Carries the title attribute `t('agent.dragHandle')` for desktop
  hover discoverability.

CSS:
- `.agent-card-grip` — `cursor: grab` (with `:active { cursor: grabbing; }`),
  `user-select: none`, `touch-action: none`, transparent background,
  ~1.25–1.5 rem square hit target, low-contrast color (e.g.
  `rgba(255,255,255,0.35)`) brightening on hover.
- `.agent-card-header` — **remove** `cursor: grab` (line 255) and
  `user-select: none` (line 256) and the `:active { cursor: grabbing; }`
  rule (lines 258–260). The header text becomes plain text again.

The header continues to have no click handler — clicks on header text
have always been silently swallowed by the existing exclusion in
`AgentGrid.svelte:118`. We update that exclusion below so header-text
clicks now open the terminal (a small UX upgrade that falls out for
free).

### 2. [`src/lib/client/components/AgentGrid.svelte`](src/lib/client/components/AgentGrid.svelte)

- Line 146: change `draggable: { handle: '.agent-card-header' }` →
  `draggable: { handle: '.agent-card-grip' }`.
- Line 118: change
  `if (target && target.closest('.agent-card-header')) return;` →
  `if (target && target.closest('.agent-card-grip')) return;`. Now
  only clicks landing on the grip are treated as drag-passthrough;
  clicks on header text or body open the terminal.
- Inline comment update to reflect that the grip is the handle.

### 3. i18n — add `agent.dragHandle`

New key (used as the grip's `title` for desktop tooltip):

- [`src/lib/i18n/de.ts`](src/lib/i18n/de.ts) — `'agent.dragHandle': 'Zum Umsortieren ziehen'`
- [`src/lib/i18n/es.ts`](src/lib/i18n/es.ts) — `'agent.dragHandle': 'Arrastrar para reordenar'`

(Phrasing to be confirmed against existing `agent.*` strings during

## Verification

1. `pnpm check` — clean.
2. `pnpm test` — clean (or fix any selector-based test that broke).
3. Dev server (`pnpm dev`) → DevTools → toggle mobile emulator (e.g.
   iPhone 14 Pro), reload `/`:
   - Vertical swipe on tile body → page scrolls.
   - Vertical swipe on tile header text → page scrolls.
   - Vertical swipe on grip icon → tile drags (gridstack reorders).
   - Click on body → terminal modal opens.
   - Click on header text → terminal modal opens (small new UX win).
4. Disable mobile emulator, repeat on desktop:
   - Click + drag from grip → tile rearranges.
   - Click + drag from header text → no drag (regression-by-design).
   - Click on body or header text → modal opens.

   `AgentGrid` component, so the fix lands in both places at once.
6. Real-device check on the user's phone via the testing host (see
   `~/.claude/projects/-home-ar-workspace-multi-agent-workbench/memory/testing_access.md`)

## Out of scope / future

- **Long-press to enter drag mode** (true scroll-vs-drag
  discrimination via timer) — would preserve "drag from anywhere on
  the header" UX on touch, but requires either a gridstack version
  with built-in delay or hand-rolling a touch interceptor. Track as
  follow-up if the grip affordance proves too small in practice.
- **Drag handles for archive/repo dashboards** — the per-repo
  dashboard at `/repos/[id]` shares `AgentGrid`, so it's covered. The
  archive page (`/repos/[id]/archive`) is a static `<table>`, not
  gridstack — no fix needed.
- **Resize handle on touch** — `alwaysShowResizeHandle: 'mobile'` is
  unrelated and stays as is. The resize handle is a separate
  micro-element in the corner; it doesn't compete with vertical
  scroll the way the full-width header did.
