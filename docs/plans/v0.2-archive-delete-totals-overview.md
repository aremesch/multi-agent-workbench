# Archive view: per-row delete, per-repo totals, cross-repo overview

## Context

The archive view at `/repos/[id]/archive` lists each repo's exited/crashed agents with time and token stats, but it is strictly read-only and single-repo. Three gaps motivate this change:

1. **No cleanup path.** Once an agent is archived, the DB row (and its worktree on disk) live forever. Users need a per-row permanent delete to reclaim disk and prune stale entries.
2. **No roll-up per repo.** Users eye-ball totals across 10+ archived agents today — the table should footer-sum time and tokens so "how much did this repo cost" is one glance.
3. **No way to compare repos.** Users have to click each repo's archive page in turn. A single overview page that compares repos side-by-side answers "which repo is eating the most time/tokens" without navigation.

## Scope

- Add a trash-icon action on every archived agent row → two-step confirm (gated by worktree dirty state) → hard DB delete + worktree removal.
- Add a totals row to the bottom of the per-repo archive table (time + tokens, summed across visible rows).
- New `/archive` page: one row per repo (only repos with ≥1 archived agent), with the same totals columns, sortable by any metric. Reached via the clickable "Archive" section label in the sidebar.

## Design decisions (from Q&A)

- **Delete always removes the worktree.** If the worktree is clean, one confirm. If dirty, a second confirm warns "uncommitted changes will be lost." Worktree-clean check happens server-side at delete time (race-safe) and is surfaced via a 409 response.
- **Overview lists only repos with archived agents** (symmetrical with the sidebar's existing Archive node).
- **Sidebar "Archive" label becomes a link** to `/archive`. Per-repo rows underneath keep linking to `/repos/[id]/archive`.

## Implementation

### 1. Delete button (per-row)

**Server — extend `DELETE /api/agents/[id]`**
- File: `src/routes/api/agents/[id]/+server.ts`
- Current behavior: kills tmux, optionally removes worktree (`?removeWorktree=1`), deletes agent row.
- Change: when `?removeWorktree=1` is set, first check the worktree's git status before removal. If `git status --porcelain` returns any lines and `?force=1` is NOT set, return HTTP `409 Conflict` with body `{ code: 'worktree_dirty', changedFiles: string[] }`. If clean OR `force=1`, proceed as today.
- Reuse: there is no existing "is worktree dirty" helper — add a small `isWorktreeDirty(worktreePath: string): Promise<{ dirty: boolean; files: string[] }>` in `src/lib/server/git/worktreeStatus.ts` (new file; uses `execFile('git', ['status', '--porcelain', '-z'])` like the existing git helpers under `src/lib/server/git/`). Skip the check if `worktree.status === 'removed'` already (already-gone = safe to delete DB row).
- Only archived statuses (`exited`, `crashed`) should permit delete. Return `409 { code: 'not_archived' }` otherwise, so a future misuse of the endpoint can't nuke a live agent. (The UI only renders the button on archived rows, so this is defense in depth.)

**Client — confirm dialog flow**
- File: `src/routes/repos/[id]/archive/+page.svelte`
- Add a trash-icon `<button>` in the existing actions cell (the column that holds "View Logs"). Inline SVG icon (codebase has no icon lib); match the button styling used for `.view-btn`.
- Use the existing `Modal.svelte` (`src/lib/client/components/Modal.svelte`) for the confirm dialog — no shadcn-svelte is vendored in this repo.
- Flow:
  1. Click trash → open confirm modal: "Permanently delete this agent and its worktree? This cannot be undone." Buttons: Cancel / Delete.
  2. On Delete → `fetch('/api/agents/<id>?removeWorktree=1', { method: 'DELETE' })`.
  3. If `204` → close modal, `invalidateAll()` to refresh the page's load data.
  4. If `409` with `code === 'worktree_dirty'` → replace modal body with a second-stage warning listing the changed files (truncate to ~10 with "+N more"). Buttons: Cancel / Delete anyway.
  5. On "Delete anyway" → `fetch('/api/agents/<id>?removeWorktree=1&force=1', { method: 'DELETE' })` → expect `204` → refresh.
  6. Any other error → surface the message inline in the modal.
- i18n: add `archive.delete`, `archive.delete.confirm`, `archive.delete.confirmDirty`, `archive.delete.confirmBtn`, `archive.delete.forceBtn`, `archive.delete.error` to `src/lib/client/i18n/*` (match the existing `archive.*` key layout).

### 2. Per-repo totals row

**Server — add totals to page load**
- File: `src/routes/repos/[id]/archive/+page.server.ts`
- After building `entries`, compute a `totals` object: sum of `totalSec`, `stats.activeSec`, `stats.idleSec`, `tokens.inputTokens`, `tokens.outputTokens`, `tokens.cacheCreationTokens`, `tokens.cacheReadTokens` across all entries. Treat `null` as 0 for sum purposes but preserve a "count of rows that contributed tokens" so the UI can render `—` when nothing contributed.
- Return as `data.totals`.

**Client — render as `<tfoot>`**
- File: `src/routes/repos/[id]/archive/+page.svelte`
- Add a `<tfoot>` after `<tbody>` with one row that spans the same 16 columns: label "Total" in the title column, dashes for role/cli/status/exit/started/ended/actions, formatted sums in the numeric columns using the existing `formatDurationHMS` and `fmtTokens` helpers.
- Style with a top border + bold to visually separate from data rows.

### 3. Cross-repo archive overview

**New route**
- Directory: `src/routes/archive/` (top-level, not under `/repos/`)
- Files: `+page.server.ts`, `+page.svelte`

**Server — `+page.server.ts`**
- Require login (`locals.user`).
- Query all repos owned by the user that have ≥1 agent with status `exited` or `crashed`. Reuse `listAgentCardsForRepo()` in a loop over the user's repos, or — cleaner — add a new query `listArchiveSummariesForUser(userId): ArchiveRepoSummary[]` in `src/lib/server/db/queries.ts` that joins `repos`, `projects`, and aggregated `agents` filtered by status.
- For each repo, aggregate exactly the same per-agent values computed today on the per-repo page (total/active/idle seconds, 4 token counts, archived agent count). Reuse `summarizeTerminalActivity()` per agent (no new heuristics) and `summarizeTokenUsage()` per agent with a `cli_session_id`. This is O(archived-agents-total); cache later if it becomes slow.
- Return `{ summaries: ArchiveRepoSummary[] }` where each summary has: `repo: { id, path, projectName }`, `archivedCount`, `totals: { totalSec, activeSec, idleSec, inputTokens, outputTokens, cacheCreationTokens, cacheReadTokens }`.

**Client — `+page.svelte`**
- Single table: columns = Project · Repo · Archived agents · Total · Active · Idle · In · Out · Cache W · Cache R · (trailing link cell).
- Each row's Project/Repo cells link to `/repos/{id}/archive` (whole row clickable is fine; keep a visible affordance).
- Add a grand-total `<tfoot>` row summing every column across repos (same pattern as the per-repo totals, for at-a-glance "how much have I spent total across everything archived").
- Empty state: if `summaries.length === 0`, show "Nothing archived yet" matching the existing `archive.noArchived` copy.
- Reuse `formatDurationHMS` and the local `fmtTokens` formatter (copy the 12-line helper or extract it to `src/lib/shared/format.ts` alongside `formatDurationHMS` — extract, since it will now have two call sites).

**Sidebar entry point**
- File: `src/lib/client/components/RepoTreeSidebar.svelte`
- Change the `<div class="section-label archive-label">` at line 120 into an `<a href="/archive">` (or wrap the text in one, preserving the count badge). Add an `active` class when `$page.url.pathname === '/archive'`. Keep the per-repo archive links under it unchanged.
- No change to breadcrumb on the per-repo archive page — the existing back-link to `/repos/<id>` stays.

## Verification

- `pnpm check` — type-clean.
- Manual, per feature:
  1. **Per-row delete, clean worktree:** archive an agent whose branch has been merged, click trash → single confirm → row disappears, disk dir gone (`ls worktrees/…`), branch gone (`git branch | grep maw/`).
  2. **Per-row delete, dirty worktree:** inside an archived agent's worktree, `echo x >> README.md` (do not commit), click trash → first confirm → expect second-stage modal listing `README.md` → "Delete anyway" → row gone, disk dir gone.
  3. **Per-row delete cancel:** click trash, dismiss — row + worktree unchanged.
  4. **Per-repo totals:** archive 2+ agents in one repo, open `/repos/<id>/archive` → tfoot sums match the visible rows (manual addition sanity check).
  5. **Cross-repo overview:** archive agents across at least 2 repos, open `/archive` (click sidebar "Archive" label) → each repo appears once with correct totals; deleting an agent from a per-repo archive page and re-visiting `/archive` shows the reduced counts.
  6. **Overview empty state:** on an account with zero archived agents, `/archive` renders the empty-state text.
  7. **Sidebar entry:** click "Archive" label → navigates to `/archive` and the label shows active style; repo items beneath still navigate to their per-repo archive.
