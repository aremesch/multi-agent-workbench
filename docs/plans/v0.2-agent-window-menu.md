# v0.2 ŌĆö Agent window menu (Show Plan / Show Log / Exit Agent)

> **Rename note:** before implementation begins, `git mv` this file to

## Context

Today an agent's modal terminal "window" only exposes the close button
and a status badge in its header. To stop a running agent, users have to
type `/exit` in the agent's CLI (or `Ctrl-C` and confirm) ŌĆö there is no
UI affordance. To inspect the persisted `terminal_log` they have to wait
for the agent to exit so it lands in `/repos/[id]/archive` and reach for
the **View logs** action there. And there is no way at all to surface a
plan file the agent has written into `<worktree>/docs/plans/`.

This plan adds a kebab (three-dot) menu to the modal header for **CLI
coding agents** (`claude-code`, `codex`, `gemini` ŌĆö not browser, not the
`shell` smoke adapter) with three items:

1. **Show Plan** ŌĆö opens a modal that renders the markdown plan file
   the agent has written into `<worktree>/docs/plans/`. The most
   recently modified candidate auto-loads; if more candidates exist a
   small switcher dropdown in the modal's header lets the user flip
   between them. Empty state if none.
2. **Show Log** ŌĆö reuses the existing archive log replay
   ([`ArchivedAgentLogModal`](../../src/lib/client/components/ArchivedAgentLogModal.svelte))
   pointed at the running agent's id. The endpoint already serves
   running agents (`terminal_log` is populated continuously) ŌĆö no
   backend work required for this item.
3. **Exit Agent** ŌĆö confirmation dialog ŌåÆ `POST /api/agents/[id]/stop`
   (existing route). Status flips to `exited` via the existing
   `SC_AgentState` WS broadcast; the dashboard auto-refreshes via the
   page's existing exit effect.

The menu is gated on `cli_kind Ōłł {claude-code, codex, gemini}`. The
shell smoke adapter and browser/browser-stream agents already have
their own purpose-built UI ("Stop session" inside `BrowserView`) and
do not get the kebab.

## Implementation

### 1. Plan-discovery helpers (`src/lib/server/plans/agentPlans.ts`)

Pure server-side module. No DB queries ŌĆö operates on file paths +
`git`.

```ts
export interface PlanFileSummary {
  name: string;        // basename, e.g. "v0.2-foo.md"
  modifiedMs: number;  // mtime in epoch ms
  sizeBytes: number;
}

export interface RenderedPlan {
  name: string;
  html: string;        // sanitized markdown ŌåÆ HTML
}

// Read <worktree>/.claude/settings.json#plansDirectory if present,
// fall back to "docs/plans". Validates the value is a relative path
// without ".." (defense in depth ŌĆö settings.json is owner-controlled
// but a checked-in worktree could ship a malicious value).
export function resolvePlansDir(worktreePath: string): string;

// List *.md files in <worktree>/<plansDir>, filtered to those
// touched by this agent's branch (added/modified relative to
// agents.base_sha, including uncommitted). Falls back to listing
// every *.md if base_sha is unreachable. Sorted by mtime desc.
//
// Uses git via execFile (not spawn shell) ŌĆö same pattern as the
// existing src/lib/server/git/agentCommits.ts helpers.
export async function listAgentPlans(
  worktreePath: string,
  plansDir: string,
  baseSha: string | null
): Promise<PlanFileSummary[]>;

// Read + render a specific plan. Path-traversal guard: filename
// must match /^[A-Za-z0-9._-]+\.md$/ ŌĆö no slashes, no leading dot.
// Returns null if the file no longer exists (race with rename).
export async function renderAgentPlan(
  worktreePath: string,
  plansDir: string,
  filename: string
): Promise<RenderedPlan | null>;
```

Internal renderer composes `marked.parse(md)` then
`DOMPurify.sanitize(html, { USE_PROFILES: { html: true } })`. Code
fences keep their `<pre><code class="language-ŌĆ”">` markup; DOMPurify
preserves `class` attributes by default.

### 2. Plan route (`src/routes/api/agents/[id]/plan/+server.ts`)

```ts
GET /api/agents/:id/plan          ŌåÆ { files: PlanFileSummary[] }
GET /api/agents/:id/plan?file=X   ŌåÆ { name, html }
```

- CSRF not required (GET).
- Auth: copy the four-line block from `stop/+server.ts` (lines 23ŌĆō27).
- 404 with `{ code: 'no_worktree' }` if `getWorktree(agent.worktree_id)`
  returns nothing.
- Empty `files: []` is a 200 ŌĆö the client renders the empty-state.
  We reserve 4xx for genuine errors (auth, missing agent, bad file
  param).
- Bad `?file=` value ŌåÆ 400 `{ code: 'invalid_filename' }`.
- File listed but missing on disk ŌåÆ 404 `{ code: 'plan_not_found' }`
  (race with `git mv`).

### 4. PlanViewerModal (`src/lib/client/components/PlanViewerModal.svelte`)

Wraps `<Modal>`. State machine:

- `loading` ŌĆö fetch in flight (initial list + per-file render).
- `empty` ŌĆö server returned `files: []`. Body shows
  `t('plan.modal.empty', { dir })` and the resolved `<plansDir>`.
- `viewing` ŌĆö auto-loaded the most recently modified file. Renders
  `{@html html}` inside a `.markdown-body` div with `prose`-ish
  styling (headings, lists, code blocks, tables ŌĆö all CSS-only, no
  extra deps).
- `error` ŌĆö fetch or render failure. Shows the error message + retry
  button.

**Multi-file switcher.** When `files.length > 1`, the modal's `<Modal>`
`headerRight` snippet renders a `<select>` populated with `name ┬Ę iso
date` entries, sorted by mtime desc. Initial value is the most recent
file (auto-picked into `viewing` on first paint). On change ŌåÆ component
re-fetches `?file=<name>` and re-renders. The select is visually small
(matches the dashboard's existing `<select class="status-filter">`
pattern) and stays out of the way for single-file plans (omitted
entirely when `files.length <= 1`).

Initial fetch flow: `GET /api/agents/[id]/plan` ŌåÆ if `files.length === 0`
render `empty`; else immediately fire `GET ?file=<files[0].name>` and
render `viewing` once it resolves. The list response and the file
response can be coalesced server-side in a future optimization (return
`{files, html}` for the most-recent file in one round-trip), but the
two-call shape is fine for v0.2 and keeps the route simple.

```ts
async function confirmExitAgent(): Promise<void> {
  if (!openAgent) return;
  exitConfirmOpen = false;
  try {
    const res = await apiFetch(`/api/agents/${encodeURIComponent(openAgent.id)}/stop`, { method: 'POST' });
    if (!res.ok && res.status !== 409) throw new Error(`HTTP ${res.status}`);
    // Status flip arrives via SC_AgentState; existing $effect at line 87
    // calls invalidateAll() once it lands, so the modal stays open until
    // the user closes it themselves.
  } catch (err) {
    exitErrorMsg = err instanceof Error ? err.message : String(err);
  }
}
```

## Security notes

- **Path traversal:** `filename` query param is validated against
  `/^[A-Za-z0-9._-]+\.md$/`. Rejects slashes, `..`, hidden files.
  Server-side `path.resolve(worktreePath, plansDir, filename)` is
  asserted to start with `path.resolve(worktreePath, plansDir) + sep`
  before any read.
- **Markdown XSS:** plan files are checked into the agent's worktree
  by the agent itself; an attacker who already has write access to a
  worktree has bigger problems than HTML injection. Even so we
  sanitize with DOMPurify (HTML profile, no SVG / MathML) so a
  malicious plan file can't `fetch()` an exfil endpoint when a logged-in
  user opens it.
- **Auth:** identical pattern to `/api/agents/[id]/log`. Owner-only.
- **No new credentials, env vars, or secrets.** `marked` and DOMPurify
  pull no native bindings ŌĆö they bundle into `build/server.js` cleanly
  and don't need to be in the esbuild `external` list.
