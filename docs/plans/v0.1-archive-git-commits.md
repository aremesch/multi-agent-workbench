# Archive drawer: per-agent git commit log

## Context

Every MAW agent runs in a dedicated worktree on branch `maw/<agentId>`. The
branch survives archival (the agent row only transitions to `exited` /
`crashed`; the branch and worktree directory are preserved unless the user
explicitly removes them on delete). Today the archive drawer shows only the
role name, status, CLI kind, and timestamps — none of the actual *work*
the agent produced is visible.

Goal: surface the commits each archived agent made on its branch, directly
inside the archive drawer, as an expandable section under each row. When
the repo has an `origin_url` that resolves to GitHub or Gitea, SHAs become
clickable links to the remote commit page. No API tokens, no network calls
from the server (beyond the local `git` invocation) — purely reading what
already exists on disk.

Out of scope: PR resolution, push status, multi-branch history, commits on
branches other than `maw/<agentId>`.

## Approach

### 1. Backend: local `git log` endpoint

Add `GET /api/agents/[id]/commits` returning commits authored on the
agent's branch within its lifetime window.

Implementation:

- **Route file**: `src/routes/api/agents/[id]/commits/+server.ts` (new).
- Load the agent row via existing queries in
  `src/lib/server/db/queries.ts`. Join to `worktrees` (for `branch` +
  parent repo path) and `repos` (for `origin_url`). Add a new query
  `getAgentCommitContext(agentId, userId)` there — returns
  `{ repoPath, branch, originUrl, startedAt, endedAt }`.
  - `repoPath` is the **parent repo** path (not the worktree),
    resolved via `worktrees.repo_id → repos.path`.
  - `startedAt` / `endedAt` come from the latest `agent_runs` row.
- Shell out to `git` via `child_process.execFile` (already used in
  `src/lib/server/git/WorktreeManager.ts` — reuse its runner if it
  exposes one, otherwise inline a minimal `execFile` wrapper):
  ```
  git -C <repoPath> log \
      --pretty=format:%H%x1f%an%x1f%aI%x1f%s%x1f%b%x1e \
      --no-merges \
      <branch> \
      ^<mergeBase with repos.default_branch>
  ```
  Using `^<mergeBase>` scopes output to commits unique to the agent's
  branch — cleaner than `--since/--until`, which would also sweep in
  unrelated commits that happen to fall in the time window. Compute
  the merge base with a prior `git merge-base <branch> <defaultBranch>`
  call; if that fails (branch deleted / never existed), return
  `{ commits: [] }` with a `reason` field.
- Parse the `\x1e`-delimited records into
  `{ sha, author, date, subject, body }[]`.
- Derive a `remote` descriptor from `origin_url`:
  - `src/lib/server/git/remoteUrl.ts` (new, small, pure): parse
    SSH (`git@host:owner/repo.git`) and HTTPS (`https://host/owner/repo(.git)?`)
    forms. Detect provider by hostname heuristic: `github.com` →
    `github`; anything else → `gitea` (MAW's current Gitea-friendly
    default; unknown hosts still get a link, just no special icon).
  - Return `{ provider, webBase }` where `webBase` is e.g.
    `https://github.com/owner/repo` or `https://gitea.example/owner/repo`.
    Commit URL = `${webBase}/commit/${sha}` (works for both GitHub and
    Gitea).
- Response shape:
  ```ts
  {
    commits: Array<{
      sha: string; shortSha: string;
      author: string; date: string;
      subject: string; body: string;
    }>;
    remote: { provider: 'github' | 'gitea' | 'unknown'; webBase: string } | null;
  }
  ```
- Enforce user ownership via existing auth helpers used by the sibling
  `DELETE /api/agents/[id]/+server.ts` route — mirror that pattern
  exactly so a user cannot read another user's commit log.

### 2. Frontend: expandable commits panel in the archive drawer

Modify **`src/lib/client/components/AgentArchiveDrawer.svelte`** only.

- Restructure each row so the existing one-line meta block becomes
  clickable (button-wrapped or a disclosure triangle on the left).
  Clicking toggles an expanded region rendered under the row.
- Lazy-fetch on first expand: track
  `commitsByAgent = $state<Record<string, CommitsResponse | 'loading' | 'error'>>({})`.
  On expand, if the entry is missing, `fetch('/api/agents/<id>/commits')`
  and populate; subsequent toggles reuse cached data. No prefetch —
  a drawer with 50 archived agents should not fire 50 requests.
- Rendering:
  - Empty result → muted `No commits on maw/<agentId>.` line.
  - Each commit: short SHA (monospace) · subject. Hovering or clicking
    the SHA opens a tooltip or inline block with author, date, and body.
    If `remote` is non-null, wrap the SHA in an `<a target="_blank" rel="noopener">`
    pointing at `${webBase}/commit/${sha}`.
- Keep the existing Delete button on the collapsed row; don't move it
  into the expanded panel.
- No changes to the load path in `src/routes/+page.server.ts` — the
  drawer's existing `archivedAgents` prop is still enough; commits are
  fetched on demand.

### 3. Types

Add `AgentCommitsResponse` + `AgentCommit` + `AgentRemote` to
`src/lib/shared/types.ts` (already the shared-safe types module).

## Critical files

- `src/routes/api/agents/[id]/commits/+server.ts` — new route.
- `src/lib/server/db/queries.ts` — add `getAgentCommitContext`.
- `src/lib/server/git/remoteUrl.ts` — new, remote URL parser.
- `src/lib/server/git/WorktreeManager.ts` — reuse its `git` runner if
  exposed, else copy the `execFile` pattern.
- `src/lib/client/components/AgentArchiveDrawer.svelte` — UI + fetch.
- `src/lib/shared/types.ts` — response types.
- Sibling reference for auth: `src/routes/api/agents/[id]/+server.ts`
  (the DELETE route).

## Verification

1. `pnpm check` clean.
2. Dev server: `pnpm dev`. Spawn a shell-adapter agent, make 2–3
   commits inside its worktree (`git -C <worktree> commit --allow-empty -m "test N"`),
   let the agent exit, open the archive drawer, expand the row:
   - Expect all 3 commits, newest first, with short SHA + subject.
   - Body visible on expand/hover.
   - `No commits` message for an agent that made none.
3. Set `repos.origin_url` to a `https://github.com/owner/repo` URL for
   one repo; confirm SHAs render as links to `.../commit/<sha>`.
   Repeat with a Gitea-style URL (`https://git.example.com/owner/repo`)
   and an SSH URL (`git@github.com:owner/repo.git`). All three should
   yield a working `webBase`.
4. Attempt to `GET /api/agents/<other-user-agent-id>/commits` as a
   different user — expect 403/404, matching the DELETE route.
5. Delete the branch (`git branch -D maw/<agentId>` in the parent repo)
   and re-open the drawer — expect a graceful empty/reason response,
   no 500.
