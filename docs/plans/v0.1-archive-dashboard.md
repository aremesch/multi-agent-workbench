# v0.1 — Archive dashboard

## Context

Archived (exited/crashed) agents currently appear two places: expanded under
the sidebar's **Archive** node, and (until the previous fix) mixed into the
per-repo dashboard. After that fix, clicking an archived repo lands on
`/repos/[id]` which now correctly hides them — so the link goes nowhere
useful. We want a dedicated post-mortem view for archived agents:

- Sidebar: **Archive** lists repos only (no per-agent expansion).
- Clicking an archived repo opens a new **archive dashboard** for that repo
  with a table of its archived agents.
- Per agent: a **View logs** button (modal, xterm replay of `terminal_log`)
  and basic statistics (total time, active vs idle time, exit reason).
- Tokens / AI vs user-time split are deferred — they need per-adapter
  instrumentation that's out of scope here. Columns are designed so that
  swap-in is non-breaking (see Future work).

## Approach

Heuristic stats from existing data, no schema changes, no runtime
instrumentation. Reuse existing components (Modal, xterm setup,
`AgentCardRow`, `terminal_log` queries).

### 1. Sidebar — drop agent expansion under Archive

`src/lib/client/components/RepoTreeSidebar.svelte`

- Remove the `openArchiveRepos` state and the per-repo disclosure toggle
  (lines ~180–189) and the nested agent `<ul>` (lines ~200–216).
- Render each archived repo as a single link to
  `/repos/[repoId]/archive` showing repo path + archived-agent count badge.
- Keep the top-level Archive collapse/expand (`archiveOpen`).

### 2. New route — `/repos/[id]/archive`

`src/routes/repos/[id]/archive/+page.server.ts` (new)

- Auth + repo ownership check (mirror `repos/[id]/+page.server.ts:55-58`).
- `const ARCHIVED_STATUSES: AgentStatus[] = ['exited', 'crashed'];`
- Fetch agents: `listAgentCardsForRepo(userId, repoId, ARCHIVED_STATUSES)`
  (`src/lib/server/db/queries.ts:290`).
- Fetch latest `agent_runs` row per agent for `started_at`/`ended_at` /
  `exit_code`/`reason` — add a small helper `getLatestRunForAgent(agentId)`
  in `queries.ts` (single `SELECT … ORDER BY started_at DESC LIMIT 1`).
- For each agent, compute heuristic stats from `terminal_log.ts`:
  - `totalSec = (ended_at ?? updated_at) - (started_at ?? created_at)`
  - `activeSec` / `idleSec` from a SQL pass that walks chunk timestamps and
    sums gaps (>30s gap counts as idle, otherwise active). One query per
    agent is fine — archive lists are small. Helper:
    `summarizeTerminalActivity(agentId, idleThresholdSec = 30)` in
    `queries.ts`, returning `{ activeSec, idleSec, firstTs, lastTs }`.
- Return `{ repo, archivedAgents: Array<AgentCardRow & { run, stats }> }`.

`src/routes/repos/[id]/archive/+page.svelte` (new)

- Page header: "Archive — <repo path>" with breadcrumb back to `/repos/[id]`.
- Table (or stacked cards on narrow viewports):
  | Role | Status | Exit reason | Started | Ended | Total | Active | Idle | Actions |
- Times rendered with a small `formatDuration(sec)` helper
  (`src/lib/shared/format.ts` — new file, since no equivalent exists).
- Actions cell: **View logs** button → opens log modal for that agent.

### 3. Log viewer modal

`src/lib/client/components/ArchivedAgentLogModal.svelte` (new)

- Props: `agentId`, `open`, `onClose`.
- On open: dynamic-import xterm (mirror `AgentTerminalPanel.svelte:1-50`),
  fetch all chunks via a new endpoint `/api/agents/[id]/log` (see below),
  feed them into the terminal in one pass, then `term.scrollToTop()`.
  No WebSocket subscription — read-only static replay.
- Reuse `Modal.svelte` shell. Size: large (full-width on mobile).

`src/routes/api/agents/[id]/log/+server.ts` (new)

- Auth, ownership check (`getAgent` + user_id match — mirror snapshot route).
- Stream concatenated chunks: query `terminal_log` for `agent_id = ?`
  ordered by `seq`, write each `chunk` BLOB to the response as
  `application/octet-stream`. Reuse `listTerminalChunksSince(agentId, 0)`
  (`queries.ts:~470`) or add a thin `listAllTerminalChunks(agentId)`
  generator if memory is a concern (archived logs are byte-budget pruned
  already, so a single buffered response is acceptable).

### 4. Layout server data

`src/routes/+layout.server.ts` — no changes needed; `archivedRepos` is
already grouped by repo with agent counts available via `agents.length`.
Sidebar will read the count directly from the existing structure.

## Verification

1. `pnpm check` — clean (0/0).
2. `pnpm dev`, log in.
3. Spawn an agent against a repo, kill it (Ctrl-C in tmux or invalidate
   snapshot) so it transitions to `exited` / `crashed`.
4. Sidebar: open Archive — confirm only repo names appear (no agent
   sub-tree), each with its archived count.
5. Click an archived repo → lands on `/repos/[id]/archive`. Confirm the
   archived agent appears in the table with non-zero total time and
   sensible active/idle split.
6. Click **View logs** → modal opens with the agent's terminal output
   replayed via xterm; ANSI colors render; scroll works; close works.
7. Spot-check `/repos/[id]` still shows only live agents (regression
   guard from the previous fix).

## Future work (out of scope)

- Real AI vs user-time split + token totals require `agent_events` table
  + per-adapter instrumentation. The table layout reserves columns so
  these can be added without UI rework.
- Bulk delete / export of archived agents.
