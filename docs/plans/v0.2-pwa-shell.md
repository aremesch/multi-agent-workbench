# v0.2 Phase A — PWA Installable Shell

## Context

MAW v0.1 is feature-complete. The original roadmap slated PWA + push
notifications as v0.2 ("the point of the project"). We're splitting v0.2
into two phases:

- **Phase A (this plan):** Make the app installable as a PWA on Android
  and desktop — manifest, service worker, offline fallback. No push yet.
- **Phase B (follow-up plan):** VAPID keys, push subscription, alert
  pipeline, notification actions, tap-to-answer.

Phase A is deliberately small so we can verify installability on a real
Android device before wiring up push.

## Approach: SvelteKit Native Service Worker (no vite-pwa)

Use SvelteKit's built-in `src/service-worker.ts` support instead of
`@vite-pwa/sveltekit`. Reasons:

1. **Zero new dependencies** — SvelteKit provides `$service-worker`
   module (`build`, `files`, `version`) out of the box.
2. **Full control for Phase B** — push/notificationclick handlers are
   just appended event listeners; no fighting a plugin config.
3. **Minimal caching needs** — the app is WebSocket-heavy; we only need
   an app-shell cache + offline fallback, not Workbox runtime caching.

## Implementation Steps

### 1. Create `static/` directory + web app manifest

**New file: `static/manifest.webmanifest`**

```json
{
  "name": "Multi-Agent Workbench",
  "short_name": "MAW",
  "description": "Manage AI coding agents from your phone",
  "start_url": "/",
  "scope": "/",
  "display": "standalone",
  "background_color": "#0a0a0a",
  "theme_color": "#0a0a0a",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" },
    { "src": "/icons/icon-512-maskable.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" }
  ]
}
```

Colors match the existing `<meta name="theme-color">` in `app.html:7`
and the CSS surface color (`#0a0a0a`).

### 2. Create placeholder icons

**New files under `static/icons/`:**

- `icon-192.png` (192x192)
- `icon-512.png` (512x512)
- `icon-512-maskable.png` (512x512, with safe-zone padding)

Generate programmatically with a script using `canvas` (Node) — dark
background `#0a0a0a`, accent text "MAW" in `#a5d6ff`. Simple and
replaceable later with real branding.

### 3. Add apple-touch-icon to `src/app.html`

Add after the favicon link (line 5):

```html
<link rel="apple-touch-icon" href="%sveltekit.assets%/icons/icon-192.png" />
```

No other changes to `app.html` — the manifest link already exists on line 8.

### 4. Create the service worker — `src/service-worker.ts`

Core responsibilities:

- **Install:** Precache all Vite build outputs (`build`) and selected
  static files (`files`), keyed by `version`.
- **Activate:** Purge stale caches, `clients.claim()`.
- **Fetch:**
  - `/_app/immutable/*` → cache-first (content-hashed filenames).
  - Navigation requests → network-first; on failure serve inline
    offline HTML.
  - `/api/*`, `/ws` → passthrough (never cache).
  - Other static files → cache-first fallback to network.

The offline fallback is an inline HTML string (no separate route) — dark
themed, "You're offline" message, retry button. This avoids auth/layout
dependencies that would break offline.

Phase B extension points are marked with comments:
```ts
// Phase B: self.addEventListener('push', ...);
// Phase B: self.addEventListener('notificationclick', ...);
```

### 5. No install prompt UI (defer to Phase A.5)

Rely on the browser's native install banner / address-bar icon. A custom
"Install App" button in the header menu can be added later if needed.

## Verification

1. `pnpm build` succeeds — SW should appear in build output.
2. `pnpm preview` (or `node build/server.js` behind HTTPS proxy):
   - DevTools → Application → Manifest: all fields parse, icons load.
   - DevTools → Application → Service Workers: registered, active.
   - Lighthouse → PWA audit: passes installability checks.
3. Android Chrome: "Add to Home Screen" prompt appears or is available
   in browser menu. App launches in standalone mode.
4. Offline test: enable airplane mode → reload → see offline fallback
   page. Reconnect → tap Retry → live app loads.
5. WebSocket not broken: open a terminal modal, verify live output still
   streams (SW does not intercept `/ws`).

## Notes

- **Dev mode:** SvelteKit does not register the SW in dev (`build` and
  `files` are empty). PWA testing requires `pnpm build && pnpm preview`
  or the production server.
- **HTTPS required:** User confirmed this is already handled via reverse
  proxy.
- **Icon script:** A one-off Node script using `canvas` to generate the
  PNGs. Can be deleted after icons are committed.
