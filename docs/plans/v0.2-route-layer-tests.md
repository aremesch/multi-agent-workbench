## Context

The route-layer follow-up promised at the close of `v0.2-vitest-unit-tests.md` Phase 10 never landed.

**Current actuals**: lines 39.25 / branches 84.54 / functions 63.69 / statements 39.25
**Current gate**:    lines 26    / branches 84    / functions 62    / statements 26
**16 uncovered `+server.ts` routes** — full list below; three already-tested templates (`plan`, `upload-image`, `claude-hook`) prove the pattern works for every shape we need.

**Goal**: lift function coverage back over 70%, then ratchet the gate up to ~`floor(actual − 1.5)` so it bites on regressions again.

## Approach

One collocated `server.test.ts` per route, mirroring [`src/routes/api/agents/[id]/upload-image/server.test.ts`](src/routes/api/agents/[id]/upload-image/server.test.ts). Mocks declared inline with `vi.mock()`, `RequestEvent` constructed by hand, handlers called directly, errors caught via the existing `expectHttpError` helper.

### Test-harness recipe (reuse verbatim)

```ts
vi.mock('$lib/server/db/queries', () => ({ /* mocked query fns */ }));
vi.mock('$lib/server/auth/csrf', () => ({ verifyCsrf: vi.fn() }));

async function call(opts: CallOpts = {}): Promise<Response> {
  const event = {
    locals: { user: opts.user === undefined ? { id: 'user-1' } : opts.user },
    params: { id: opts.id ?? 'agent-1' },
    request: opts.request,
    url: opts.url ?? new URL('http://localhost/api/...'),
    cookies: { get: () => undefined }
  };
  return HANDLER(event as unknown as Parameters<typeof HANDLER>[0]);
}
```

For routes that depend on `locals.supervisor`, pass a mock `{ kill: vi.fn(), … }` via `locals`. For routes that hit subprocess wrappers (`Tmux`, `WorktreeManager`, fs helpers) `vi.mock()` the whole module — no need to thread anything through `locals`.

### Per-route coverage matrix

For every route, cover the applicable subset:

- **401** when `locals.user` is null
- **403** when `verifyCsrf` throws (mutating routes only)
- **404** when DB query returns a row owned by a different user (or no row)
- **Happy path** asserting Response status + body shape
- **400** for each zod / manual validation branch the route exposes

Skip cases that don't apply (GET routes don't need CSRF; user-preference routes have no row-owner concept).

- `pnpm check` cleanup — the 22 strict-mode test errors on this worktree pre-date this work; CI doesn't gate on it.
- Re-including page/layout routes in coverage — they stay excluded.
- Refactoring routes for testability without flagging the change to the user first; if a route is genuinely hard to test, mark a single branch `it.todo` with a one-line reason and ship the rest.
- Adding new e2e specs — pure unit-test ratchet.
