# Image / Screenshot Insertion into the Agent Terminal

## Context

A daily friction-point compared to Claude Desktop: there is no way to
hand an agent a screenshot. In Claude Desktop the user pastes
(Cmd/Ctrl+V) and the image rides along with the next message. In MAW
the agent runs as a CLI inside tmux, so neither browser-clipboard
binary paste nor terminal drag-drop can cross the wire.

The goal is feature parity â€” and a touch better, because we get drag-drop
and a mobile-friendly button for free. claude-code, codex, and gemini
all accept a literal file path in the prompt body (claude-code via the
`@<path>` mention syntax, with a `[Image #N]` echo in v2.0.73+;
codex / gemini accept the same `@<path>` form). So the plumbing is:

1. Browser receives an image (paste / drop / file picker).
2. POST it to a new MAW route that writes it inside the agent's
   worktree at `.maw/uploads/<timestamp>-<rand>.<ext>`.
3. Inject ` @.maw/uploads/<file> ` into the running agent via the
   existing `send_keys` WebSocket path.
4. The CLI sees the path on next submit and attaches the image.

No protocol bump: the upload is plain HTTPS; the keystroke uses v6
`CS_SendKeys` unchanged.

5 MB and the four MIMEs come straight from the claude-code limits (so
the upload won't surprise the CLI). MIME is taken from the multipart
declaration â€” magic-byte sniffing is a follow-up if abuse appears.

### New route â€” [src/routes/api/agents/[id]/upload-image/+server.ts](src/routes/api/agents/[id]/upload-image/+server.ts)

```
POST /api/agents/:id/upload-image
Body: multipart/form-data, single field `file` (Blob)
200: { relativePath, filename, sizeBytes, mime }
400: { code: 'mime' | 'size' | 'no_file' }
401 / 403 / 404: standard
501: { code: 'cli_kind_unsupported' }
```

Handler order:

1. `if (!locals.user) error(401)`.
2. `getAgent(params.id)` â†’ 404; `agent.user_id !== locals.user.id` â†’ 403.
3. `if (!isCodingCliKind(agent.cli_kind)) error(501)` â€” defense in depth even if the client renders the button on the wrong adapter.
4. `getWorktree(agent.worktree_id)` â†’ 404.
5. `await request.formData()`, assert `file instanceof File`.
6. `validateUpload(file.type, file.size)` â€” early-out *before* `arrayBuffer()` so we never load oversized blobs into memory.
7. `bytes = new Uint8Array(await file.arrayBuffer())`.
8. `await writeAgentImage(wt.path, file.type, bytes)`.
9. `return json({...})`.

This is the **first** multipart route in the codebase; SvelteKit's
default CSRF guard already covers same-origin POSTs.

## Client side â€” [src/lib/client/components/AgentTerminalPanel.svelte](src/lib/client/components/AgentTerminalPanel.svelte)

One internal helper, three input surfaces.

```ts
async function uploadAndInjectImage(file: File): Promise<void> {
  // 1. mirror server validation client-side for fail-fast UX
  // 2. POST /api/agents/<id>/upload-image (multipart)
  // 3. on 200: getMawWsClient().sendKeys(agent.id, ` @${relativePath} `)
  //    leading + trailing space preserves word boundaries with whatever
  //    the user has already typed in the prompt
  // 4. term?.focus()
  // 5. toast: t('agentTerminal.image.toastInjected', { filename, path })
}
```

Render gate (derived):
`imagesEnabled = isCodingCliKind(agent.cli_kind) && agent.adapterFeatures?.acceptsImageAttachment`.
The flag is delivered via the existing `cliKinds` payload that already
ships `mobileQuickKeys`, so no new query.

### Three input surfaces

- **Paste** â€” bind `on:paste` to the modal root (xterm's host element
  doesn't fire paste for blobs). Iterate `e.clipboardData.items`,
  pick `kind === 'file' && type.startsWith('image/')`,
  `e.preventDefault()` *only in that branch* so plain text paste keeps
  flowing through xterm.
- **Drop** â€” `on:dragover|preventDefault`, `on:drop` filtering
  `e.dataTransfer.files` to `image/*`. Track `dragenter`/`dragleave`
  depth to drive a translucent overlay (`agentTerminal.image.dropOverlay`).
- **Paperclip button** â€” hidden `<input type="file"
  accept="image/png,image/jpeg,image/gif,image/webp">` plus a labelled
  icon button rendered next to the mobile quick-keys row at lines
  173â€“190. Loops on `change` and reuses the helper.

All three exit through `uploadAndInjectImage`, so error-handling and
the toast are in one place.

## Non-goals / deferred

- **Recently-attached thumbnail strip** â€” defer to a v1.5 follow-up.
  The toast plus the literal `@<path>` echo in the terminal is enough
  confirmation for the first cut.
- **GC / janitor** â€” no auto-cleanup. Files persist with the worktree
  and don't get committed thanks to `.maw/.gitignore`. Users can
  `rm -rf .maw/uploads/` manually. If `.maw/uploads/` becomes a real
  disk-pressure problem, add a janitor in a later plan.
- **Magic-byte MIME sniffing** â€” trust the multipart MIME for v1;
  revisit if abuse surfaces. Note that `isCodingCliKind` already gates
  this to authenticated, owner-scoped requests, so the blast radius is
  the user's own worktree.
- **Per-image alt-text / caption UI** â€” not yet. Users can type freely
  before/after the injected `@<path>`.
