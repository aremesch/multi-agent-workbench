# Inline Project / Role / Repo Creation During Agent Spawning

## Context

The spawn form currently requires roles and repos to exist before a user can spawn an agent. If either is missing, it shows a warning and links away to separate full-page creation forms. This forces navigation away from the dashboard and breaks the "spawn an agent in one flow" goal. This plan adds inline creation of roles, projects, and repos directly within the spawn modal — no page navigation required.

---

## Approach

Three thin JSON API routes handle creation and return the new entity's data. The `SpawnAgentForm` Svelte component calls them with `fetch()`, updates its local reactive state, and auto-selects the new item. The existing `use:enhance` spawn flow is completely untouched.

---

## Files to Create

### `src/routes/api/projects/+server.ts`
- `POST` — body: `{ name, default_branch? }`
- Auth check (`locals.user`), validate name required, branch valid (`/^[\w./-]+$/`)
- Calls `insertProject` (reuses same validation as `projects/new/+page.server.ts`)
- Returns `200 { id, name, default_branch }` or `400 { error }`

### `src/routes/api/roles/+server.ts`
- `POST` — body: `{ name, cli_kind, system_prompt? }`
- Validates name required, cli_kind in `locals.supervisor.registry.list()`
- Calls `insertRole` with `default_args_json: '{}'`, `tool_config_json: '{}'`, `repo_scope_json: '{}'`
- Returns `200 { id, name, cli_kind }` or `400 { error }`

### `src/routes/api/repos/+server.ts`
- `POST` — body: `{ project_id, path, origin_url? }`
- Verifies project exists and belongs to `locals.user`
- Performs full git validation (same logic as `projects/[id]/repos/new/+page.server.ts`):
  - Empty dir → `WorktreeManager.initEmpty(path, project.default_branch)`
  - Non-empty non-git → reject
  - Existing git → `WorktreeManager.ensureDefaultBranch(path, project.default_branch)`, handle `no_master` case
- Calls `insertRepo`
- Returns `200 { id, path, projectName }` or `400/403 { error }`

### `src/lib/client/components/SpawnAgentForm.svelte` (main change)

**New interfaces:**
```typescript
export interface SpawnProjectOption { id: string; name: string; default_branch: string }
export interface CliKindOption { kind: string; displayName: string }
```

**New props** (added to destructure):
```typescript
projects: SpawnProjectOption[];
cliKinds: CliKindOption[];
```

**New reactive state:**
```typescript
let roleOptions = $state<SpawnRoleOption[]>([...roles]);
let repoOptions = $state<SpawnRepoOption[]>([...repos]);
let projectOptions = $state<SpawnProjectOption[]>([...projects]);
let selectedRoleId = $state(roles[0]?.id ?? '');
let selectedRepoId = $state(repos[0]?.id ?? '');

// Inline role creation
let showNewRole = $state(false);
let newRoleName = $state('');
let newRoleCliKind = $state(cliKinds[0]?.kind ?? '');
let newRoleError = $state<string | null>(null);
let savingRole = $state(false);

// Inline repo creation
let showNewRepo = $state(false);
let newRepoProjectId = $state(projects[0]?.id ?? '');
let newRepoPath = $state('');
let newRepoOriginUrl = $state('');
let newRepoError = $state<string | null>(null);
let savingRepo = $state(false);

// Nested inline project creation (inside repo form)
let showNewProject = $state(false);
let newProjectName = $state('');
let newProjectBranch = $state('main');
let newProjectError = $state<string | null>(null);
let savingProject = $state(false);
```

**Three `fetch()` handler functions:**

`createRole()` — POST `/api/roles`, on success push to `roleOptions`, set `selectedRoleId`, collapse form, reset fields.

`createProject()` — POST `/api/projects`, on success push to `projectOptions`, set `newRepoProjectId`, collapse nested project form, reset fields.

`createRepo()` — POST `/api/repos`, on success push to `repoOptions`, set `selectedRepoId`, collapse repo form, reset all repo/project fields.

**Markup layout** (remove the old `{#if roles.length === 0}` gate entirely — form always shown):

```
Role:   [select bind:value={selectedRoleId}]  [+ New role]
        {#if showNewRole}
          inline form: Name + CLI kind select + error + [Cancel][Create role]

Repo:   [select bind:value={selectedRepoId}]  [+ New repo]
        {#if showNewRepo}
          Project: [select bind:value={newRepoProjectId}]  [+ New project]
                   {#if showNewProject}
                     inline form: Name + Branch + error + [Cancel][Create project]
          Path: [input]
          Origin URL: [input, optional]
          error
          [Cancel]  [Add repo] (disabled while showNewProject)

Task title (optional): [input]
Task body (optional): [textarea]
error
[Cancel]  [Spawn] (disabled while showNewRole OR showNewRepo OR !selectedRoleId || !selectedRepoId)
```

**Selects** use `bind:value` + `name` attribute — both work simultaneously in SvelteKit; form submission picks up the bound value.

## Edge Cases

| Issue | Mitigation |
|---|---|
| "Add repo" clicked before project saved | `disabled={showNewProject \|\| savingRepo}` on Add repo button |
| Stale error state on re-open | Toggle onclick and Cancel onclick reset `*Error = null` and clear fields |
| `noUncheckedIndexedAccess` on array index | `x[0]?.id ?? ''` pattern throughout |
| 28rem width limit | `font-size: 0.85rem` on inline forms, reduced padding |
| `no_master` branch case in repos API | Convert `WorktreeManager.ensureDefaultBranch` result to JSON 400 |
| `json()` helper needed | `import { json } from '@sveltejs/kit'` in all three API routes |
